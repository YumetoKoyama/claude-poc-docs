# 非機能要件

> ID 凡例: [docs/凡例.md](../凡例.md) 参照
> 未確定のベースライン項目は本文に TBD を残さず `オープン課題.md` の `Q-NF*` へ切り出し、本ファイルからは参照のみとする。

## 性能

| 項目 | 要件 | 備考 |
|------|------|------|
| 想定同時アクセス数 | 最大 100 ユーザー | 要件素材どおり |
| 想定データ規模（主要エンティティの件数） | 案件（ENT-003）最大 300 件（1 社平均 3 案件想定） | データモデル.md の ENT-003 |
| 主要画面の応答時間 | 1 秒以内を目標 | 要件素材どおり |
| バッチの実行時間枠 | 深夜 2:00〜4:00（Q-NF9 決定済み） | 該当バッチは `外部インターフェース一覧.md` の EXT-010（夜間バッチ） |

## 可用性

| 項目 | 要件 |
|------|------|
| 稼働時間帯 | 24 時間稼働を目標とする（Q-NF9 決定済み）。ただしデモ用途のため目標稼働率としては明示しない（下記のとおり対象外） |
| 計画停止 | 実施を許容する。周知方法は「工事中」画面（SCR-019）および通知機能での事前案内とする（Q-NF9 決定済み） |
| 目標稼働率 | **明示しない**（デモ用途想定であることを要件素材で明示済みのため、本項目は「対象外」として確定済みとする） |
| RTO（目標復旧時間） | **明示しない**（同上、確定済み「対象外」） |
| RPO（目標復旧時点） | **明示しない**（同上、確定済み「対象外」） |

## セキュリティ

| 項目 | 要件 |
|------|------|
| 認証方式 | ログイン ID・パスワードによる認証（`functional/認証.md` 参照） |
| 認可ポリシー | テナント単位のデータ境界必須。テナント内のロール分離は第 1 版では行わない（`権限マトリクス.md` 参照） |
| パスワードポリシー（最小桁数 / 文字種 / 有効期間 / ロックアウト） | 最小 8 文字以上、英字（大文字・小文字問わず）と数字を各 1 文字以上含む。有効期間による強制変更は行わない。ログイン失敗ロックは連続 5 回失敗で 15 分間ロック（Q-NF4・Q-NF2 決定済み） |
| セッション管理（有効時間 / 同時ログイン制御） | アイドル 30 分でタイムアウト、最大セッション長 8 時間。同時ログイン制御は第 1 版では行わない（Q-NF5 決定済み） |
| 通信暗号化（HTTPS 必須性 / 証明書運用） | HTTPS を必須とする（証明書運用は設計フェーズの責務）。対象ブラウザは直近 2 メジャーバージョンの Chrome / Edge / Safari（Q-NF6 決定済み） |
| CSRF / XSS 対策方針 | 設計フェーズの責務とし、要件レベルでは「入力サニタイズ・CSRF 対策を必須とする」方針のみを明示する |
| PII（個人情報）取り扱い方針 | 企業担当者の氏名・電話番号・メールアドレス、個人ドライバーの連絡先を扱う。連絡先の段階開示方針は BR-020 を参照。ログ・通知への個人情報混入を避ける方針とする |

## ログ・監査

| 項目 | 要件 |
|------|------|
| 監査対象操作 | 詳細な操作ログの個別確認機能は第 1 版では持たない（要件素材どおり、概要.md 対象外） |
| ログ保管期間 | 90 日間（Q-NF10 決定済み） |
| アプリケーションログレベル方針 | 設計フェーズの責務 |
| 集計指標（取得する KPI 一覧） | 個人情報に抵触しない指標（例: 合意件数）を蓄積する（要件素材どおり） |

## 運用・障害対応

| 項目 | 要件 |
|------|------|
| バックアップ（対象データ / 頻度 / 世代数） | 対象: DB 全データ。頻度: 日次（1 日 1 回）。世代数: 7 世代（1 週間分）保持（Q-NF7 決定済み） |
| 障害検知方法 | アプリケーションの死活監視（ヘルスチェック）（Q-NF8 決定済み） |
| 障害通知方法（宛先 / チャネル） | 運用担当者へメール通知（メール送信失敗・バッチ失敗の継続検知も対象、Q-NF8 決定済み） |
| 問い合わせ受付方法 | 「工事中」画面（SCR-019）で案内する。本格的な問い合わせフォームは次期対応（要件素材どおり） |
| 計画停止時の周知方法 | 「工事中」画面（SCR-019）および通知機能での事前案内（Q-NF9 決定済み） |

## アクセシビリティ・ブラウザ要件

| 項目 | 要件 |
|------|------|
| 対象ブラウザ | 直近 2 メジャーバージョンの Google Chrome / Microsoft Edge / Safari（Q-NF6 決定済み） |
| 対象デバイス（PC / モバイル / タブレット） | PC（デスクトップ）ブラウザのみを正式対応とする。モバイル・タブレットでの表示崩れは保証しない（レスポンシブ対応は第 1 版では必須としない、Q-NF6 決定済み） |
| アクセシビリティ準拠レベル（WCAG 2.x 等） | 第 1 版では WCAG 準拠を必須要件としない（将来対応、Q-NF6 決定済み）。ブランド方針・トーンは `ブランドガイドライン.md`（Q-BR1 決定済み）を参照 |
| キーボード操作要件 | 第 1 版では必須としない（将来対応、Q-NF6 決定済み） |
| スクリーンリーダー対応要件 | 第 1 版では必須としない（将来対応、Q-NF6 決定済み） |

## データ保持

| 項目 | 要件 |
|------|------|
| 取引データの保持期間 | 成約済〜評価済の案件は無期限保存（将来期間制限を追加予定、BR-023） |
| 論理削除データの物理削除タイミング | 論理削除から 24 時間後、夜間バッチ（EXT-010）で物理削除（BR-022） |
| ログ・監査記録の保持期間 | 90 日間（Q-NF10 決定済み） |

> 関連バッチ（物理削除バッチ等）は `外部インターフェース一覧.md` の EXT-010 を参照。

## サポート・利用停止

| 項目 | 要件 |
|------|------|
| 不具合報告受付 | 「工事中」画面（SCR-019）での案内に留める（本格的な問い合わせフォームは次期対応） |
| 利用停止申請機能 | 第 1 版では扱わない（要件素材どおり） |
