# 用語集

> ID 凡例: [docs/凡例.md](../凡例.md) 参照

## 用語一覧

| 用語 | 英語表記（参考） | 定義 |
|------|--------|------|
| 配送依頼企業 | Requesting Company | 配送を依頼する企業。旧称「荷主」。特定業態（大手物流企業等）に固定しない汎用的な呼称として本書で統一する。 |
| 運送会社 | Carrier Company | 配送依頼企業が登録した案件に応募し、実際に運送を行う企業。 |
| 企業アカウント（テナント） | Tenant | 1 企業に対して発行される単位。配下に複数ユーザーを持つ。配送依頼企業・運送会社のいずれも同じ仕組みで企業アカウントを持つ。 |
| ユーザー | User | 企業アカウント配下で個別にログイン ID を発行される利用者。第 1 版では同一企業アカウント内のユーザーは全員同権限。 |
| 案件 | Job | 配送依頼企業が登録する配送依頼の単位（積荷 1 件分）。ステータス（募集中〜評価済）を持つ。 |
| 応募 | Bid | 運送会社が 1 案件に対して行う 1 提案。1 社が同一案件に複数提案することはない。 |
| セット応募 | Set Bid | 運送会社が複数案件をまとめて 1 つの提案として応募する形態。同一の配送依頼企業が登録した案件同士に限る。 |
| 交渉 | Negotiation | 応募発生後、配送依頼企業と運送会社が連絡機能で金額・条件をやり取りする段階。 |
| 最終条件提示 | Final Offer | 交渉中にどちらか一方が行う、合意成立に向けた最終条件の提示操作。 |
| 合意 | Agreement | 最終条件提示に対し相手が「合意する」を行うことで成立する状態。合意成立と同時に成約処理が行われる。 |
| 成約 | Contract Execution | 合意成立時点で行われる処理。合意内容がスナップショットとして保存され、他応募がクローズされる。本書では「成約」に表記を統一し、「契約成立」「確定」「合意完了」等の表記は用いない。 |
| 成約スナップショット | Contract Snapshot | 成約処理の時点で保存される、合意金額・場所日時・トラック種別・双方の企業/担当者情報・合意時点のメッセージ情報等の複製データ。以後編集不可。 |
| 運送開始報告 | Departure Report | 運送会社が運送を開始したことを報告する操作。案件ステータスを「成約済」から「運送中」に遷移させる。 |
| 完了報告 | Completion Report | 運送会社が運送完了を報告する操作。 |
| 完了確認 | Completion Confirmation | 配送依頼企業が完了報告の内容を確認し「完了確認」を行う操作。案件ステータスを「運送中」から「完了」に遷移させる。 |
| 評価 | Rating | 完了確認後、双方が相手に対して行う★評価。一度登録すると編集不可。 |
| 通知 | Notification | 画面右上の通知アイコンを通じて表示される、成約・応募・メッセージ・ステータス更新等のお知らせ。 |
| 取引履歴 | Transaction History | 成約済〜評価済に至った案件の一覧・詳細を振り返るための記録。無期限保存（第 1 版）。 |
| 論理削除 | Logical Deletion | 削除操作の直後、データを物理的には残したまま「削除済み」として扱う状態。第 1 版では 1 日間保持する。 |
| 物理削除 | Physical Deletion | 論理削除から一定期間（24 時間）経過後、夜間バッチにより実データを削除する処理。 |

## ステータス定義（正典）

> 案件ステータスの正典は本表とする。他資料（業務ルール.md・各機能要件・データモデル.md・画面一覧.md）は本表の名称・意味と完全一致させること。**「キャンセル」は第 1 版では基本使用しない（将来用）ため、他資料でキャンセルを前提としたエラー条件・不成立条件として参照してはならない。**

| ステータス名 | 説明 | 第 1 版での利用 |
|------------|------|--------------|
| 募集中 | 案件登録直後。誰でも応募可能。 | 利用する |
| 交渉中 | 1 社以上が応募し、金額・条件の交渉が行われている。 | 利用する |
| 成約済 | 双方が最終条件提示・合意の 2 ステップで合意成立し、契約が成立した状態。 | 利用する |
| 運送中 | 運送会社が運送開始を報告した状態。 | 利用する |
| 完了 | 運送会社が完了報告を行い、配送依頼企業が完了確認した状態。 | 利用する |
| 評価済 | 双方の評価登録が完了した状態。 | 利用する |
| キャンセル | 成約前のキャンセル状態。 | **第 1 版では基本使用せず、将来拡張用として定義のみ行う。第 1 版の業務フロー・画面・通知・エラー条件からは参照しない。** |

応募ステータスの定義は `データモデル.md` の状態遷移（ST-1xx 系）を参照する。
