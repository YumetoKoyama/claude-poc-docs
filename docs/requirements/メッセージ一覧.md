# メッセージ一覧

> ID 凡例: [docs/凡例.md](../凡例.md) 参照

区分は「エラー / 警告 / 情報 / 確認 / バリデーション / 通知」のいずれかを付与する。

**文言トーン方針（Q-MSG1 決定済み）**: 敬体（です・ます調）で統一し、利用者を非難しない客観的な表現とする（例:「間違えています」ではなく「入力に誤りがあります」）。以下の全メッセージは本方針に沿った内容として確定する。
