# 機能要件: 案件登録

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 機能概要

- 機能 ID: —（関連ユースケース UC-007、UC-008 を参照）
- 利用者: 配送依頼企業ユーザー
- 目的: 配送を依頼したい積荷（案件）の情報を登録し、運送会社からの応募を受け付けられる状態にする。また自社が登録した案件の一覧・詳細・応募状況を確認できるようにする
- 関連画面（[画面 ID](../画面一覧.md)）: SCR-007（案件登録画面）、SCR-006（配送依頼企業ダッシュボード・遷移元）、SCR-008（案件一覧画面・配送依頼企業）、SCR-009（案件詳細画面・配送依頼企業）
- 関連ユースケース（[UC-XXX](../ユースケース図.md)）: UC-007（案件登録）、UC-008（案件一覧・詳細閲覧・配送依頼企業）
- 関連業務フロー（ACT-XXX）: ACT-001（案件成約フロー）
- 関連業務ルール（[業務ルール ID](../業務ルール.md)）: BR-011（交渉中への遷移）、BR-019（編集制限）

> 参照: 画面 ID は [画面一覧.md](../画面一覧.md)、ユースケース ID は [ユースケース図.md](../ユースケース図.md)、業務ルール ID は [業務ルール.md](../業務ルール.md)、業務フロー ID は [activities/](../activities/) 配下の該当ファイルを参照。

## ユーザーストーリー

### ストーリー 1 — 案件の新規登録（優先度: P1）

配送依頼企業の担当者として、運びたい荷物の情報（from/to・時刻・容積・属性・希望金額等）を入力して案件を登録したい。なぜなら、登録しないと運送会社に見つけてもらえず応募が発生しないため。

**独立テスト**: 案件登録項目一式を入力し登録操作を行うと、案件がステータス「募集中」で作成され、自社の案件一覧に表示されることを確認する。

---

### ストーリー 2 — 自社案件の一覧・詳細確認（優先度: P1）

配送依頼企業の担当者として、自社が登録した案件を「募集中（未受注）」「交渉中・済」に分けて一覧し、詳細で応募状況・連絡履歴を確認したい。なぜなら、複数案件を並行して管理し、対応漏れを防ぎたいため。

**独立テスト**: 複数ステータスの案件が存在する状態で一覧画面を開くと、ステータス区分どおりに案件が振り分けられ、詳細画面で応募運送会社一覧・連絡履歴が表示されることを確認する。

## 成功基準

- **SC-001**: 案件登録項目の入力から登録完了まで、利用者が単一の画面遷移フロー内で完結できる。
- **SC-002**: 24 時間以内に更新があった案件が一覧上で「new」として判別できる。
- **SC-003**: 自社が登録した案件のみが一覧・詳細に表示され、他社（他テナント）の案件情報は表示されない。

## 入力（ユーザーが与える情報）

| 項目名 | 型 / 形式 | 必須 | 取りうる値 | 説明 |
|-------|---------|------|----------|------|
| from 場所 | 文字列 / 住所 | 必須 | — | 積み地 |
| from 時刻（希望日時） | 日時 | 必須 | — | 出発希望日時 |
| to 場所 | 文字列 / 住所 | 必須 | — | 卸し地 |
| to 時刻（希望日時） | 日時 | 必須 | — | 到着希望日時 |
| 容積 | 数値 | 必須 | 単位: 立方メートル（m3、Q-DM3 決定済み） | 積荷の容積 |
| 属性（危険物等） | 複数選択 | 任意 | 危険物 / 要冷蔵 / 要冷凍 / 割れ物（Q-DM2 決定済み） | 特別な取り扱いが必要な属性 |
| 物品種別 | 単一選択 | 必須 | 一般貨物 / 精密機器 / 食品（冷蔵・冷凍） / 危険物 / その他（Q-DM2 決定済み） | 運ぶ物品の種類 |
| 希望トラック種別 | 単一選択 | 必須 | 軽トラック / 2t トラック / 4t トラック / 10t トラック / トレーラー（Q-DM2 決定済み） | 必要なトラックの種類 |
| 希望金額 | 数値 | 必須 | 単位: 日本円（税別）、整数円単位（Q-DM3 決定済み） | 配送依頼企業が希望する金額 |
| 備考 | 文字列 | 任意 | — | 自由記述 |

## 出力（システムが返す情報）

| 項目名 | 型 / 形式 | 説明 |
|-------|---------|------|
| 案件 ID | 内部識別子 | 登録された案件の一意識別子 |
| 案件ステータス | ST-001〜ST-007（データモデル.md 参照） | 登録直後は「募集中」（ST-001） |
| new 表示フラグ | 真偽値 | 24 時間以内の更新有無 |

## 受け入れ条件

### 正常系

- [ ] AC-001: Given 案件登録項目をすべて有効な値で入力した状態 / When 登録操作を行う / Then 案件がステータス「募集中」で作成され一覧に表示される
- [ ] AC-002: Given 自社が複数ステータスの案件を保有する状態 / When 一覧画面を開く / Then 「募集中（未受注）」「交渉中・済」に区分されて表示される

### 異常系

- [ ] AC-101: Given 必須項目が未入力、または from 時刻が to 時刻より後になっている等の不整合がある状態 / When 登録操作を行う / Then MSG-001 が表示され登録は行われない

### 境界値・特殊条件

- [ ] AC-201: Given 案件が 24 時間以内に更新された状態 / When 一覧画面を開く / Then 当該案件に new 表示がされる

### 権限境界（該当する場合）

- [ ] AC-301: Given 自社（自テナント）以外が登録した案件 / When 一覧・詳細へのアクセスを試みる / Then 一覧には表示されず、詳細への直接アクセスは拒否される（テナント越境防止）

### エッジケース

- AC-401: ステータスが「成約済」以降の案件は編集・削除の導線自体を非表示にする（BR-019、詳細は `案件削除.md` を参照）。

## 前提条件（Assumptions）

- 案件登録は配送依頼企業ユーザーのみが行い、運送会社が代理登録することはない。
- 容積・希望金額の単位は m3・日本円（税別）に確定済み（Q-DM3 決定済み）。
- 物品種別・希望トラック種別・属性の選択肢はマスタとして確定済み（Q-DM2 決定済み、初期値は `移行要件.md` MIG-002 で投入）。
