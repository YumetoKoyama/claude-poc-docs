# 機能要件: 応募

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 機能概要

- 機能 ID: —（関連ユースケース UC-010〜UC-013 を参照）
- 利用者: 運送会社ユーザー
- 目的: 募集中案件を検索・閲覧し、単体またはセットで応募する。成約前であれば応募内容を編集できるようにする
- 関連画面（[画面 ID](../画面一覧.md)）: SCR-014（募集中案件一覧画面）、SCR-015（案件詳細画面・運送会社）、SCR-015-01（セット応募選択モーダル）、SCR-016（交渉中・済一覧画面）
- 関連ユースケース（[UC-XXX](../ユースケース図.md)）: UC-010（募集中案件検索・閲覧）、UC-011（応募する）、UC-012（セット応募する）、UC-013（応募内容編集）
- 関連業務フロー（ACT-XXX）: ACT-001（案件成約フロー）
- 関連業務ルール（[業務ルール ID](../業務ルール.md)）: BR-004（応募の一意性）、BR-005（セット応募の対象範囲）、BR-006（セット応募の成立条件）、BR-007（セット内個別案件の並行応募）、BR-009（応募上限）、BR-010（応募自動締切）、BR-021（競合表示方針）

> 参照: 画面 ID は [画面一覧.md](../画面一覧.md)、ユースケース ID は [ユースケース図.md](../ユースケース図.md)、業務ルール ID は [業務ルール.md](../業務ルール.md)、業務フロー ID は [activities/](../activities/) 配下の該当ファイルを参照。

## ユーザーストーリー

### ストーリー 1 — 募集中案件の検索・閲覧（優先度: P1）

運送会社の担当者として、募集中の案件を条件（都道府県・時間帯等）で絞り込み・検索して閲覧したい。なぜなら、自社の運行計画に合う案件を効率よく見つけたいため。

**独立テスト**: 複数の募集中案件が存在する状態で絞り込み条件を指定すると、条件に合致する案件のみが一覧表示されることを確認する。

---

### ストーリー 2 — 単体応募（優先度: P1）

運送会社の担当者として、気になる案件に対して希望金額に同意するか独自の金額を提示して応募したい。なぜなら、条件が合えば早期に受注機会を確保したいため。

**独立テスト**: 募集中案件に対して応募操作を行うと、応募が登録され案件ステータスが「交渉中」に遷移することを確認する。

---

### ストーリー 3 — セット応募（優先度: P2）

運送会社の担当者として、同一の配送依頼企業が登録した複数案件をまとめて 1 つの提案として応募したい。なぜなら、複数案件を同時受注できる場合に限り運びたいという条件を表現したいため。

**独立テスト**: 同一配送依頼企業の複数案件を選択してセット応募すると、各案件の応募枠が 1 つずつ消費され、セット内すべてが合意成立して初めて成約する（オール・オア・ナッシング）ことを確認する。

---

### ストーリー 4 — 応募内容の編集（優先度: P3）

運送会社の担当者として、まだ成約していない自社の応募内容（提示金額等）を編集したい。なぜなら、交渉の途中で条件を調整したいため。

**独立テスト**: 成約前の自社応募に対して編集操作を行うと、内容が更新され相手に反映されることを確認する。

## 成功基準

- **SC-001**: 運送会社の担当者が絞り込み条件の指定から応募完了まで、他システムへの問い合わせなしに完結できる。
- **SC-002**: 応募上限（20 社）・自動締切（from 時刻の 2 時間前）に達した案件へは応募操作自体が行えない状態になる。
- **SC-003**: セット応募において、セット内 1 件でも不成立になった場合、利用者はセット全体が不成立になったことを通知経由で把握できる。

## 受け入れ条件

### 正常系

- [ ] AC-001: Given 募集中の案件が応募上限・締切に達していない状態 / When 希望金額に同意する、または独自金額を入力して応募する / Then 応募が登録され初回応募であれば案件ステータスが「交渉中」に遷移する

### 異常系

- [ ] AC-101: Given 対象案件の応募数が既に 20 社に達している、または from 時刻の 2 時間前を過ぎている状態 / When 応募操作を行う / Then MSG-004（上限超過）または MSG-005（締切超過）が表示され応募できない
- [ ] AC-102: Given 自社が既に同一案件へ応募済みの状態 / When 再度応募操作を行う / Then MSG-017 が表示され重複応募できない（BR-004）

### 境界値・特殊条件

- [ ] AC-201: Given 同一の配送依頼企業が登録した複数の募集中・交渉中案件を選択した状態 / When セット応募操作を行う / Then 各案件の応募枠が 1 つずつ消費され、1 つのセット応募として登録される（BR-005, BR-006, BR-007, BR-009）
- [ ] AC-202: Given 異なる配送依頼企業の案件を同時に選択しようとした状態 / When セット対象に追加しようとする / Then UI 上追加できないよう制御される（BR-005）

### 権限境界（該当する場合）

- [ ] AC-301: Given 自社以外（他テナント）の応募 / When 応募内容の編集を試みる / Then 拒否される（自社の応募のみ編集可能）

### エッジケース

- AC-401: セット応募のうち一部案件が他社に先に成約された場合、当該セット応募は自動的に不成立となり、セット内の他案件への応募もすべて連動してクローズされる（BR-006, BR-015）。

## 前提条件（Assumptions）

- 「希望金額に同意」を選択した場合は、独自金額入力なしに応募が成立する（要件素材の想定どおり）。
- 競合表示（他社の金額）はリロード時に最新化され、リアルタイム Push は行わない（BR-021）。
- セット応募の対象を異なる配送依頼企業の案件へ拡張することは将来課題とし、第 1 版のスコープ外とする。
