# 機能要件: 案件削除

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 機能概要

- 機能 ID: —（関連ユースケース UC-009 を参照）
- 利用者: 配送依頼企業ユーザー
- 目的: 募集中・交渉中の案件を編集・削除できるようにし、削除時は応募していた運送会社へ通知した上で論理削除→物理削除の流れでデータを扱う
- 関連画面（[画面 ID](../画面一覧.md)）: SCR-009（案件詳細画面・配送依頼企業）
- 関連ユースケース（[UC-XXX](../ユースケース図.md)）: UC-009（案件編集・削除）
- 関連業務フロー（ACT-XXX）: ACT-005（案件削除フロー・論理削除→物理削除）
- 関連業務ルール（[業務ルール ID](../業務ルール.md)）: BR-019（編集制限）、BR-022（案件削除の範囲と保持）

> 参照: 画面 ID は [画面一覧.md](../画面一覧.md)、ユースケース ID は [ユースケース図.md](../ユースケース図.md)、業務ルール ID は [業務ルール.md](../業務ルール.md)、業務フロー ID は [activities/](../activities/) 配下の該当ファイルを参照。

## ユーザーストーリー

### ストーリー 1 — 募集中・交渉中案件の編集（優先度: P2）

配送依頼企業の担当者として、成約前の案件内容（条件・希望金額等）を編集したい。なぜなら、登録後に条件を見直したい場合があるため。

**独立テスト**: 募集中の案件の内容を編集すると、変更内容が保存され運送会社側の表示にも反映されることを確認する。

---

### ストーリー 2 — 案件の削除（論理削除→物理削除）（優先度: P2）

配送依頼企業の担当者として、不要になった募集中・交渉中の案件を削除したい。なぜなら、誤登録や依頼自体の取りやめに対応したいため。

**独立テスト**: 募集中・交渉中の案件を削除すると、応募していた運送会社に通知が送られ、24 時間後の夜間バッチで物理削除されることを確認する。

## 成功基準

- **SC-001**: 削除操作の対象は「募集中」「交渉中」のみに制限され、成約済以降の案件では削除導線自体が表示されない。
- **SC-002**: 削除時、応募していた運送会社全員に通知が届く。
- **SC-003**: 論理削除から 24 時間経過後、夜間バッチにより自動的に物理削除される。

## 入力（ユーザーが与える情報）

| 項目名 | 型 / 形式 | 必須 | 取りうる値 | 説明 |
|-------|---------|------|----------|------|
| 編集後の案件内容 | 案件登録.md の入力項目に準ずる | 編集時必須 | — | UC-009 |
| 削除確認操作 | ボタン押下 | — | 削除する / キャンセル | UC-009 |

## 出力（システムが返す情報）

| 項目名 | 型 / 形式 | 説明 |
|-------|---------|------|
| 論理削除フラグ | 真偽値 | 削除操作直後に ON |
| 物理削除結果 | イベント | 夜間バッチ（EXT-010）による実行結果 |

## 受け入れ条件

### 正常系

- [ ] AC-001: Given 案件ステータスが「募集中」または「交渉中」の状態 / When 削除操作を行う / Then 論理削除され、応募していた運送会社へ通知（MSG-013）が送信される（BR-022）

### 異常系

- [ ] AC-101: Given 案件ステータスが「成約済」「運送中」「完了」「評価済」のいずれかの状態 / When 編集・削除を試みる / Then 操作導線が表示されず実行できない（BR-019）

### 境界値・特殊条件

- [ ] AC-201: Given 論理削除されてから 24 時間未満の状態 / When 一覧・詳細を確認する / Then 配送依頼企業側には「削除済み」として一覧・詳細に表示を残し、運送会社側には非表示とする（Q-J8 決定済み）

### 権限境界（該当する場合）

- [ ] AC-301: Given 自社以外（他テナント）の案件 / When 編集・削除を試みる / Then 拒否される

### エッジケース

- AC-401: 論理削除から 24 時間経過後、夜間バッチ（EXT-010）により物理削除が実行される。紐づく応募（ENT-004）・連絡メッセージ（ENT-006）は同時に物理削除し、通知（ENT-009）は物理削除の対象とせず保持する（Q-DM4 決定済み）。

## 前提条件（Assumptions）

- 削除（論理削除）から物理削除までの保持期間は 24 時間固定とする（要件素材どおり）。
- 復元機能（論理削除からの取消）は第 1 版では提供しない（スコープ外、Q-J9 決定済み）。
