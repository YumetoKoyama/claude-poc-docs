## ユーザーストーリー

### ストーリー 1 — 自社の取引履歴閲覧（優先度: P3）

配送依頼企業・運送会社どちらの担当者としても、過去に成約した案件の一覧・詳細を振り返りたい。なぜなら、過去の取引条件や相手企業の情報を参照して次の意思決定に活かしたいため。

**独立テスト**: 成約済〜評価済の過去案件が存在する状態で取引履歴一覧を開くと、自社が当事者だった案件のみが表示され、詳細で成約スナップショットの内容が確認できることを確認する。

## 成功基準

- **SC-001**: 取引履歴には自社が当事者（配送依頼企業 or 運送会社）だった案件のみが表示され、一覧の応答時間は 1 秒以内（非機能要件.md の主要画面応答時間目標に準拠）を目標とする。他社案件の混入率は 0%（自他判定 100% 正確）とする。
- **SC-002**: 取引履歴の保存期間は第 1 版では無期限であり、過去の案件が意図せず一覧から消えない。表示件数の目安は最大 300 件（データモデル.md の ENT-003 想定件数と整合）とする。

## 受け入れ条件

### 正常系

- [ ] AC-001: Given 自社が当事者の成約済〜評価済案件が存在する状態 / When 取引履歴一覧を開く / Then 該当案件が一覧表示され、詳細で成約スナップショットが確認できる

### 異常系

- [ ] AC-101: Given 取引履歴が 1 件も存在しない状態 / When 取引履歴一覧を開く / Then 空状態（該当データなし）のメッセージが表示される

### 境界値・特殊条件

- 該当なし（本機能は無期限保存の閲覧専用機能であるため境界値は発生しない）

### 権限境界（該当する場合）

- [ ] AC-301: Given 自社が当事者ではない案件 / When 取引履歴として取得を試みる / Then 表示されない

### エッジケース

- AC-401: 物理削除された案件（募集中・交渉中のうちに削除されたもの）は、そもそも成約に至っていないため取引履歴には含まれない。

## 前提条件（Assumptions）

- 取引履歴の保存期間は第 1 版では無期限とし、将来的に期間制限を追加する予定（BR-023）。
