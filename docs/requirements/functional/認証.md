# 機能要件: 認証

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 機能概要

- 機能 ID: —（関連ユースケース UC-004〜UC-006 を参照）
- 利用者: 配送依頼企業ユーザー、運送会社ユーザー
- 目的: 発行済みのログイン ID・パスワードで安全にログイン／ログアウトし、パスワードを失念した場合に再設定できるようにする
- 関連画面（[画面 ID](../画面一覧.md)）: SCR-001（ログイン画面）、SCR-003（パスワードリセット申請画面）、SCR-004（パスワードリセット再設定画面）
- 関連ユースケース（[UC-XXX](../ユースケース図.md)）: UC-004（ログイン）、UC-005（ログアウト）、UC-006（パスワードリセット）
- 関連業務フロー（ACT-XXX）: なし（単一操作の完結する機能のため業務アクティビティ図は作成しない）
- 関連業務ルール（[業務ルール ID](../業務ルール.md)）: BR-002（ユーザー単位ログイン ID 必須）

> 参照: 画面 ID は [画面一覧.md](../画面一覧.md)、ユースケース ID は [ユースケース図.md](../ユースケース図.md)、業務ルール ID は [業務ルール.md](../業務ルール.md)を参照。

## ユーザーストーリー

### ストーリー 1 — ログイン（優先度: P1）

登録済みユーザーとして、自分のログイン ID とパスワードでシステムにログインしたい。なぜなら、ログインしないと自社の案件・応募情報にアクセスできないため。

**独立テスト**: 有効なログイン ID・パスワードでログイン操作を行うと、自テナントのダッシュボードへ遷移することを確認する。

---

### ストーリー 2 — ログアウト（優先度: P1）

ログイン中のユーザーとして、操作終了時に明示的にログアウトしたい。なぜなら、共用端末等でセッションが残り続けることを防ぐため。

**独立テスト**: ログアウト操作を行うと、セッションが終了しログイン画面に戻り、戻る操作等で保護された画面に再アクセスできないことを確認する。

---

### ストーリー 3 — パスワードリセット（優先度: P2）

パスワードを忘れたユーザーとして、登録済みメールアドレス宛に再設定用リンクを受け取り、新しいパスワードを設定したい。なぜなら、パスワード忘失時にサポートへ問い合わせずに自力で復旧したいため。

**独立テスト**: パスワードリセットを申請すると再設定用メールが送信され、リンク経由で新しいパスワードを設定後、新パスワードでログインできることを確認する。

## 成功基準

- **SC-001**: 有効な認証情報でのログインが即時（体感 1 秒程度）で完了し、ダッシュボードへ遷移する。
- **SC-002**: ログイン失敗が連続 5 回に達した場合、簡易ロック機構により以降 15 分間ログイン試行が制限される（Q-NF2 決定済み）。
- **SC-003**: パスワードリセット申請から新パスワード設定完了までを、メール受信さえできれば他者への問い合わせなしに完了できる。

## 入力（ユーザーが与える情報）

| 項目名 | 型 / 形式 | 必須 | 取りうる値 | 説明 |
|-------|---------|------|----------|------|
| ログイン ID | 文字列 | 必須 | — | UC-004 |
| パスワード | 文字列 | 必須 | — | UC-004 |
| 登録メールアドレス | 文字列（メール形式） | 必須 | — | UC-006（リセット申請時） |
| 新パスワード | 文字列 | 必須 | 最小 8 文字以上、英字（大文字・小文字問わず）と数字を各 1 文字以上含む（Q-NF4 決定済み） | UC-006（再設定時） |

## 出力（システムが返す情報）

| 項目名 | 型 / 形式 | 説明 |
|-------|---------|------|
| 認証結果 | 成功 / 失敗 | ログイン可否 |
| セッション | 内部トークン | ログイン中の状態を保持する。有効時間はアイドル 30 分でタイムアウト、最大セッション長 8 時間（Q-NF5 決定済み） |
| リセットメール送信結果 | 成功 / 失敗 | EXT-001（パスワードリセットメール送信）の結果 |

## 受け入れ条件

### 正常系

- [ ] AC-001: Given 発行済みの有効なログイン ID・パスワードを持つ状態 / When ログイン操作を行う / Then 認証に成功し自テナントのダッシュボードへ遷移する
- [ ] AC-002: Given ログイン中の状態 / When ログアウト操作を行う / Then セッションが終了しログイン画面へ遷移する

### 異常系

- [ ] AC-101: Given ログイン ID またはパスワードが誤っている状態 / When ログイン操作を行う / Then MSG-015 が表示されログインできない
- [ ] AC-102: Given 登録されていないメールアドレスでパスワードリセットを申請した状態 / When 申請操作を行う / Then 送信有無を推測されないよう画面上は成功時と同一の案内を表示する（アカウント存在の露見防止）

### 境界値・特殊条件

- [ ] AC-201: Given ログイン失敗が連続 5 回に達した状態（Q-NF2 決定済み） / When さらにログインを試行する / Then 簡易ロック機構により 15 分間ログインを拒否し MSG-016 を表示する

### 権限境界（該当する場合）

- 第 1 版は同権限のみ（BR-003）のため、ログイン後にアクセスできる機能範囲はテナント種別（配送依頼企業 / 運送会社）にのみ依存し、テナント内でのロール差はない。

### エッジケース

- AC-401: パスワードリセット URL の有効期限（発行から 1 時間、Q-NF3 決定済み）切れ後にリンクへアクセスした場合、MSG-019 を表示し再度リセット申請を促す。

## 前提条件（Assumptions）

- 二要素認証・アカウント承認フローは将来対応であり、第 1 版では単純な ID・パスワード認証のみとする。
- 同時ログイン制御（同一ユーザーの多重ログイン許可可否）は第 1 版では制限しない（同一ユーザーの複数端末同時ログインを許可する、Q-NF5 決定済み）。

## 関連未解決事項

Q-NF2（ログイン失敗ロック）・Q-NF3（パスワードリセット URL 有効期限）・Q-NF4（パスワードポリシー）・Q-NF5（セッション有効時間・同時ログイン制御）はいずれも closed 済み（`オープン課題.md` 参照）。決定内容は本ファイルの SC-002・入力/出力表・AC-201・AC-401・前提条件に反映済み。本機能に紐づく未解決事項は現時点で無い。
