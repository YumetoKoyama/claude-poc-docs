# 機能要件: 運送ステータス報告

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 機能概要

- 機能 ID: —（関連ユースケース UC-017〜UC-019 を参照）
- 利用者: 運送会社ユーザー、配送依頼企業ユーザー
- 目的: 成約後の運送実施状況（運送開始・完了）を運送会社が報告し、配送依頼企業が完了内容を確認して案件を完了させる
- 関連画面（[画面 ID](../画面一覧.md)）: SCR-009（案件詳細画面・配送依頼企業）、SCR-015（案件詳細画面・運送会社）
- 関連ユースケース（[UC-XXX](../ユースケース図.md)）: UC-017（運送開始報告）、UC-018（完了報告）、UC-019（完了確認）
- 関連業務フロー（ACT-XXX）: ACT-002（運送実施フロー）
- 関連業務ルール（[業務ルール ID](../業務ルール.md)）: BR-017（成約後のステータス遷移）、BR-019（編集制限）

> 参照: 画面 ID は [画面一覧.md](../画面一覧.md)、ユースケース ID は [ユースケース図.md](../ユースケース図.md)、業務ルール ID は [業務ルール.md](../業務ルール.md)、業務フロー ID は [activities/](../activities/) 配下の該当ファイルを参照。

## ユーザーストーリー

### ストーリー 1 — 運送開始報告（優先度: P1）

運送会社の担当者として、実際に運送を開始したタイミングでステータスを報告したい。なぜなら、配送依頼企業に進捗を可視化したいため。

**独立テスト**: 成約済みの案件で運送開始報告を行うと、案件ステータスが「運送中」に遷移することを確認する。

---

### ストーリー 2 — 完了報告と完了確認（優先度: P1）

運送会社の担当者として運送完了を報告し、配送依頼企業の担当者として内容を確認して完了を確定させたい。なぜなら、双方の確認をもって取引完了とみなし、以後の評価フェーズに進みたいため。

**独立テスト**: 運送会社が完了報告を行った後、配送依頼企業が完了確認操作を行うと、案件ステータスが「完了」に遷移することを確認する。

## 成功基準

- **SC-001**: 運送会社の 1 回の操作で運送開始・完了の各ステータスが即座に反映される。
- **SC-002**: 配送依頼企業が完了報告の内容を確認してから完了確認するまでの操作が案件詳細画面内で完結する。

## 入力（ユーザーが与える情報）

| 項目名 | 型 / 形式 | 必須 | 取りうる値 | 説明 |
|-------|---------|------|----------|------|
| 運送開始報告操作 | ボタン押下 | — | — | UC-017 |
| 完了報告操作 | ボタン押下 | — | — | UC-018 |
| 完了確認操作 | ボタン押下 | — | — | UC-019 |

| 項目名 | 型 / 形式 | 説明 |
|-------|---------|------|
| 案件ステータス | ST-003〜ST-005（データモデル.md 参照） | 成約済 → 運送中 → 完了 |

## 受け入れ条件

### 正常系

- [ ] AC-001: Given 案件ステータスが「成約済」の状態 / When 運送会社が運送開始報告を行う / Then 案件ステータスが「運送中」に遷移する（BR-017）
- [ ] AC-002: Given 案件ステータスが「運送中」で運送会社が完了報告済みの状態 / When 配送依頼企業が完了確認操作を行う / Then 案件ステータスが「完了」に遷移する（BR-017）

### 異常系

- [ ] AC-101: Given 案件ステータスが「成約済」でない状態 / When 運送開始報告を行おうとする / Then 操作導線が表示されず報告できない
- [ ] AC-102: Given 運送会社が完了報告を行っていない状態 / When 配送依頼企業が完了確認を行おうとする / Then 完了確認操作は表示・実行できない

### 境界値・特殊条件

- 該当なし（本機能は一方向のステータス遷移のみを扱うため境界値は BR-019 の編集制限側に整理する）

### 権限境界（該当する場合）

- [ ] AC-301: Given 自社が当事者ではない案件 / When 運送開始報告・完了報告・完了確認を試みる / Then 拒否される

### エッジケース

- AC-401: 運送開始・完了報告は報告後の取消不可とする（Q-J4 決定済み）。誤操作時は連絡機能で相手と調整する。完了確認前の差戻し機能は第 1 版では提供せず、内容に相違がある場合は連絡機能で調整のうえ完了確認の実施要否を担当者が判断する（Q-J5 決定済み）。

## 前提条件（Assumptions）

- 運送開始報告・完了報告は運送会社側の自己申告であり、GPS 等による位置情報連携は第 1 版では行わない。
- 完了確認が長期間行われない場合の督促・自動確認は第 1 版では実施しない（Q-J6 決定済み、将来課題）。

## 関連未解決事項

Q-J4（運送開始・完了報告の取消可否）・Q-J5（完了確認前の差戻し要否）・Q-J6（完了確認が長期間行われない場合の督促要否）はいずれも closed 済み（`オープン課題.md` 参照）。決定内容は本ファイルの AC-401・前提条件に反映済み。本機能に紐づく未解決事項は現時点で無い。
