# 機能要件: 交渉合意成約

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 機能概要

- 機能 ID: —（関連ユースケース UC-014〜UC-016 を参照）
- 利用者: 配送依頼企業ユーザー、運送会社ユーザー
- 目的: 応募後の金額・条件交渉を連絡機能で行い、最終条件提示と合意の 2 ステップで成約処理（スナップショット保存・他応募クローズ）まで完結させる
- 関連画面（[画面 ID](../画面一覧.md)）: SCR-009（案件詳細画面・配送依頼企業）、SCR-015（案件詳細画面・運送会社）
- 関連ユースケース（[UC-XXX](../ユースケース図.md)）: UC-014（連絡・金額交渉）、UC-015（最終条件提示）、UC-016（合意する）
- 関連業務フロー（ACT-XXX）: ACT-001（案件成約フロー）
- 関連業務ルール（[業務ルール ID](../業務ルール.md)）: BR-006（セット応募の成立条件）、BR-008（セット応募の一括合意）、BR-012（合意成立の要件）、BR-013（成約時スナップショット保存）、BR-014（他応募者クローズ通知）、BR-015（セット応募クローズの連動）、BR-016（成約後キャンセル不可）、BR-020（連絡先表示の段階制御）

> 参照: 画面 ID は [画面一覧.md](../画面一覧.md)、ユースケース ID は [ユースケース図.md](../ユースケース図.md)、業務ルール ID は [業務ルール.md](../業務ルール.md)、業務フロー ID は [activities/](../activities/) 配下の該当ファイルを参照。

## ユーザーストーリー

### ストーリー 1 — 連絡・金額交渉（優先度: P1）

配送依頼企業・運送会社どちらの担当者としても、定型文または自由入力メッセージで相手とやり取りし、金額・条件を詰めたい。なぜなら、応募時点の金額のままでは折り合わないことが多く、交渉の余地が必要なため。

**独立テスト**: 応募後の案件詳細画面でメッセージを送信すると、相手側にも同じ内容が表示され、連絡履歴として蓄積されることを確認する。

---

### ストーリー 2 — 最終条件提示と合意（優先度: P1）

交渉中の担当者として、条件がまとまったら「最終条件を提示」し、相手が「合意する」を選ぶことで正式に成約させたい。なぜなら、口頭やメッセージだけでは合意の境界が曖昧になり、後のトラブルの元になるため。

**独立テスト**: 最終条件提示後に相手が合意操作を行うと、案件ステータスが「成約済」に遷移し、合意内容がスナップショットとして保存され、他の応募者がクローズされることを確認する。

## 成功基準

- **SC-001**: 最終条件提示から合意までの 2 ステップだけで成約処理（スナップショット保存・他応募クローズ・通知）がすべて自動的に完了する。
- **SC-002**: 成約と同時に、当該案件への他の応募者全員に「他社に決定しました」通知が届く。
- **SC-003**: セット応募が成約した場合、セット内すべての案件が一括で合意成立扱いになる（個別合意は発生しない）。

## 出力（システムが返す情報）

| 項目名 | 型 / 形式 | 説明 |
|-------|---------|------|
| 連絡履歴 | メッセージのリスト | 案件・応募単位で時系列表示。各メッセージに送信者のユーザー名（アカウント登録.md で登録した識別名）を表示する |
| 成約スナップショット | データ一式（ENT-007） | 合意金額・場所日時・トラック種別・双方の企業/担当者情報・合意時点のメッセージ情報 |
| 通知（MSG-006、MSG-007） | 文字列 | 他応募者クローズ通知、セット連動不成立通知 |

## 受け入れ条件

### 正常系

- [ ] AC-001: Given 交渉中の案件・応募 / When 一方が最終条件を提示し相手が合意する / Then 案件ステータスが「成約済」に遷移し、成約スナップショットが保存され、他の応募者がクローズされ通知される（BR-012, BR-013, BR-014）
- [ ] AC-002: Given セット応募が交渉中の状態 / When セット一括で最終条件提示・合意が行われる / Then セット内すべての案件が一括で成約済みとなる（BR-008）

### 異常系

- [ ] AC-101: Given 成約済み以降の案件・応募 / When メッセージ送信や最終条件提示を試みる / Then 操作が拒否される（BR-016, BR-019 と整合）

### 境界値・特殊条件

- [ ] AC-201: Given セット応募のうちいずれか 1 案件が他社に先に成約された状態 / When 当該セット応募の状態を確認する / Then セット応募全体が自動的に不成立となり、セット内の他案件への応募も連動してクローズされる（BR-006, BR-015）

### 権限境界（該当する場合）

- [ ] AC-301: Given 自社が当事者ではない案件・応募 / When 連絡・最終条件提示・合意操作を試みる / Then 拒否される

### エッジケース

- AC-401: 最終条件提示後は取消・再提示不可（合意 or 破棄のみ、Q-J2 決定済み）。合意フェーズ進行中（最終条件提示済み・未合意）に同一案件の別応募が先に成約した場合、当該提示中の応募も BR-014 と同様に自動的にクローズし「他社に決定しました」通知を送付する。提示中であった最終条件提示は無効化され、以後の合意操作は拒否される（Q-J3 決定済み）。

## 前提条件（Assumptions）

- 連絡機能の相手連絡先表示は、相手企業の電話番号・メールアドレスは募集中・交渉中でも表示可能、個人ドライバー等の個人連絡先は成約済み以降のみ表示する（BR-020）。
- 成約後のキャンセル機能は第 1 版では提供しない（BR-016）。
