# 画面設計: SCR-002 企業アカウント登録画面

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 画面概要

- 画面 ID: SCR-002
- URL パス: `/register`
- 対応ユースケース: UC-001（企業アカウント登録・配送依頼企業）, UC-002（同・運送会社）
- 関連機能要件: `docs/requirements/functional/アカウント登録.md`
- 所属レイアウト: 認証前独立レイアウト

## 入力

| 項目名 | 型 | 必須 | 送信先（operationId.フィールド） | 説明 |
|-------|----|------|--------------------------|------|
| 企業種別 | 単一選択 | 必須 | `registerTenant.tenantType` | 配送依頼企業 / 運送会社 |
| 法人名 | 文字列 | 必須 | `registerTenant.companyName` | |
| 住所 | 文字列 | 必須 | `registerTenant.address` | |
| 会社電話番号 | 文字列 | 必須 | `registerTenant.companyPhone` | |
| 会社メールアドレス | 文字列（メール形式） | 必須 | `registerTenant.companyEmail` | |
| 支払方法 | 単一選択 | 必須 | `registerTenant.paymentMethod` | 請求書払い / 銀行振込 |
| 利用規約・プライバシーポリシー同意 | チェック | 必須 | `registerTenant.termsAgreed` | |
| ユーザー名（識別名） | 文字列 | 必須 | `registerTenant.initialUser.displayName` | |
| ログイン ID | 文字列 | 必須 | `registerTenant.initialUser.loginId` | |
| 登録メールアドレス | 文字列（メール形式） | 必須 | `registerTenant.initialUser.email` | |
| ログイン用パスワード | 文字列 | 必須 | `registerTenant.initialUser.password` | 最小8文字、英字+数字各1文字以上（Q-NF4） |

## 出力・表示内容

| 項目名 | 型 | データ源（operationId.フィールド） | 説明 |
|-------|----|--------------------------|------|
| 登録結果メッセージ | 文字列 | `registerTenant` の 201/400/409 | 成功時は SCR-001 へ遷移し案内表示。失敗時は下記バリデーションメッセージ |

## バリデーションメッセージ

| 項目名 | 条件 | メッセージ |
|-------|------|-----------|
| 利用規約同意 | 未同意で登録操作 | MSG-018: 利用規約・プライバシーポリシーへの同意が必要です。 |
| 各入力項目 | 必須未入力・メール形式不正 | MSG-014: 必須項目が未入力、またはメールアドレス等の形式に誤りがあります。内容を確認してください。 |
| 法人名+住所 | 既存テナントと完全一致（正規化後、Q-A1） | 重複エラー（`ErrorResponse.code=DUPLICATE`）。既存テナント管理者への連絡を案内する文言を表示（救済手段なし、Q-A2） |

## 画面遷移（この画面から）

| トリガー | 遷移先（画面 ID） | 条件 |
|---------|--------------------|------|
| 登録成功 | SCR-001 | `registerTenant` が 201 |
| キャンセル / 戻る | SCR-001 | — |

## API 対応

| operationId | HTTP メソッド | パス | 概要 |
|-------------|-------------|------|------|
| registerTenant | POST | /api/tenants | 企業アカウント＋初回ユーザー登録 |

## セキュリティ考慮事項

- 認証: 不要
- 認可: なし（未認証で実行、権限マトリクス.md UC-001/UC-002「未認証状態で実行」）
- CSRF 対策: JWT 方式のため主要リスク低減
- その他: パスワードは `x-sensitive: true`（レスポンスに含めない）。法人重複判定は DB UNIQUE 制約で最終防御（tables/tenants.md）
