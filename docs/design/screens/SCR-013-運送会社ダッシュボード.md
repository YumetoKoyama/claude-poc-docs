# 画面設計: SCR-013 運送会社ダッシュボード

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 画面概要

- 画面 ID: SCR-013
- URL パス: `/dashboard`（運送会社）
- 対応ユースケース: UC-010（募集中案件検索・閲覧、サマリ表示）, UC-021（通知確認）
- 関連機能要件: `docs/requirements/functional/応募.md`, `docs/requirements/functional/通知.md`
- 所属レイアウト: 共通シェル

## 入力

なし（表示専用画面）。

## 出力・表示内容

| 項目名 | 型 | データ源（operationId.フィールド） | 説明 |
|-------|----|--------------------------|------|
| 募集中案件サマリ件数 | 数値 | `getJobStatusSummary(scope=recruiting).counts[]` | 全テナント公開の募集中案件件数 |
| 自社応募状況 | 数値の配列 | `listMyBids.items[].status` を集計（画面側で集計、またはサーバー側集計を将来拡張） | 交渉中/成約/クローズの件数目安 |
| new 表示件数 | 数値 | `getJobStatusSummary(scope=recruiting).newCount` | |
| 未読通知件数 | 数値 | `getUnreadNotificationCount.unreadCount` | |

## API 対応

| operationId | HTTP メソッド | パス | 概要 |
|-------------|-------------|------|------|
| getJobStatusSummary | GET | /api/jobs/status-summary?scope=recruiting | 募集中案件サマリ取得 |
| listMyBids | GET | /api/bids | 自社応募状況取得 |
| getUnreadNotificationCount | GET | /api/notifications/unread-count | 未読通知件数取得 |

## セキュリティ考慮事項

- 認証: 必要
- 認可: 運送会社ユーザーのみ到達（配送依頼企業ユーザーは SCR-006 へ誘導）
- CSRF 対策: JWT Bearer 方式
- その他: `listMyBids` は自社（carrierTenantId）フィルタを必ず適用

## 受け入れ条件

> ダッシュボードは独自の機能要件ファイルを持たず、応募.md・通知.md の既存 AC を集約表示する。

| AC-ID | 区分 | Given（前提状態） | When（操作） | Then（期待結果） | 関連 BR | operationId |
|-------|------|-----------------|------------|----------------|--------|-------------|
| 応募/AC-001 | 正常系 | 募集中の案件が存在する状態 | ダッシュボードを開く | 募集中案件のサマリ件数が表示される | — | getJobStatusSummary |
| 通知/AC-001 | 正常系 | 未読通知が存在する状態 | 通知アイコンを確認する | 未読件数が表示される（9件超は「9+」） | BR-024 | getUnreadNotificationCount |
