# 画面設計: SCR-014 募集中案件一覧画面（運送会社）

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 画面概要

- 画面 ID: SCR-014
- URL パス: `/jobs/recruiting`
- 対応ユースケース: UC-010（募集中案件検索・閲覧）, UC-011（応募する）
- 関連機能要件: `docs/requirements/functional/応募.md`
- 所属レイアウト: 共通シェル

## 入力

| 項目名 | 型 | 必須 | 送信先（operationId.フィールド） | 説明 |
|-------|----|------|--------------------------|------|
| 検索・絞り込み条件（自由入力） | 文字列 | 任意 | `listJobs.query` | 都道府県名等を含む fromLocation/toLocation の文字列一致 |
| 時間帯絞り込み（下限） | 日時 | 任意 | `listJobs.fromDatetimeAfter` | |
| 時間帯絞り込み（上限） | 日時 | 任意 | `listJobs.fromDatetimeBefore` | |
| ソート順 | 単一選択 | 任意 | `listJobs.sort` | |
| 応募金額 | 数値 | 応募時必須 | `createBid.amount` | 希望金額に同意 or 独自金額 |
| セット対象案件選択 | 複数選択 | セット応募時必須 | `createSetBid.items[]`（SCR-015-01 経由） | |

## 出力・表示内容

| 項目名 | 型 | データ源（operationId.フィールド） | 説明 |
|-------|----|--------------------------|------|
| 募集中案件一覧 | リスト | `listJobs(scope=recruiting, status=RECRUITING).items[]` | requesterCompanyName 等（BR-020: 会社名・連絡先は募集中でも表示可） |
| 応募締切 | 日時 | `listJobs.items[].bidDeadline` | fromDatetime の2時間前（BR-010）。締切超過は応募ボタン非活性 |
| 応募数 | 数値 | `listJobs.items[].bidCount` | 件数のみ（金額は非表示） |
| new 表示 | 真偽値 | `listJobs.items[].isNew` | |
| ページング情報 | オブジェクト | `listJobs.pageMeta` | |

## バリデーションメッセージ

| 項目名 | 条件 | メッセージ |
|-------|------|-----------|
| 応募操作 | 応募上限（20社）到達 | MSG-004: この案件は応募上限（20社）に達しているため応募できません。 |
| 応募操作 | 自動締切超過（from時刻の2時間前） | MSG-005: 応募受付は終了しました（希望日時の2時間前を過ぎています）。 |
| 応募操作 | 既に応募済み | MSG-017: この案件には既に応募済みです。 |
| セット対象選択 | 異なる配送依頼企業の案件を選択 | MSG-003: 異なる配送依頼企業の案件を同じセットに含めることはできません。 |

## 画面遷移（この画面から）

| トリガー | 遷移先（画面 ID） | 条件 |
|---------|--------------------|------|
| 案件行クリック | SCR-015 | 対象 `jobId` を渡す |
| 「セット応募」操作 | SCR-015-01 | セット対象選択モーダルを起動 |

## API 対応

| operationId | HTTP メソッド | パス | 概要 |
|-------------|-------------|------|------|
| listJobs | GET | /api/jobs?scope=recruiting | 募集中案件一覧取得 |
| createBid | POST | /api/jobs/{jobId}/bids | 応募 |

## セキュリティ考慮事項

- 認証: 必要
- 認可: 運送会社ユーザーのみ応募可（配送依頼企業は 403）。募集中案件自体は全運送会社テナントに公開（権限マトリクス.md 5節）
- CSRF 対策: JWT Bearer 方式
- その他: 検索クエリは `pattern: '^[^<>]*$'` で XSS 対策

## 受け入れ条件

| AC-ID | 区分 | Given（前提状態） | When（操作） | Then（期待結果） | 関連 BR | operationId |

| AC-001 | 正常系 | 募集中の案件が応募上限・締切に達していない状態 | 希望金額に同意する、または独自金額を入力して応募する | 応募が登録され初回応募であれば案件ステータスが「交渉中」に遷移する | BR-011 | createBid |
