# 画面設計: SCR-009-01 セット応募連動確認モーダル（配送依頼企業向け）

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 画面概要

- 画面 ID: SCR-009-01
- URL パス: SCR-009 上のモーダル（独立 URL を持たない）
- 対応ユースケース: UC-016（合意する。セット応募の一括承認）
- 関連機能要件: `docs/requirements/functional/交渉合意成約.md`, `docs/requirements/functional/応募.md`
- 所属レイアウト: 共通シェル（SCR-009 上のオーバーレイ）

## 入力

| 項目名 | 型 | 必須 | 送信先（operationId.フィールド） | 説明 |
|-------|----|------|--------------------------|------|
| セット一括承認操作 | ボタン | — | `agreeSetBid`（パラメータなし、`setBidId` はモーダル起動元から） | |

## 出力・表示内容

| 項目名 | 型 | データ源（operationId.フィールド） | 説明 |
|-------|----|--------------------------|------|
| セット内対象案件一覧 | リスト | `getSetBidById.bids[].job`（jobId, fromLocation, toLocation, desiredAmount 等） | 案件ごとに個別合意はしないことを明示 |
| セット内各応募の状態 | リスト | `getSetBidById.bids[].status` | いずれかが CLOSED_* の場合は一括承認不可を表示 |
| セット状態 | 文字列 | `getSetBidById.status` | |

## バリデーションメッセージ

| 項目名 | 条件 | メッセージ |
|-------|------|-----------|
| セット一括承認 | セット内いずれかが既に他状態へ遷移 | MSG-009: 相手の状態が変化したため、この操作は完了できません。最新の状態を確認してください。 |

## 画面遷移（この画面から）

| トリガー | 遷移先（画面 ID） | 条件 |
|---------|--------------------|------|
| 一括承認成功 | SCR-009 | `agreeSetBid` が 200。モーダルを閉じ案件詳細を再取得 |
| キャンセル | SCR-009 | モーダルを閉じる |

## API 対応

| operationId | HTTP メソッド | パス | 概要 |
|-------------|-------------|------|------|
| getSetBidById | GET | /api/set-bids/{setBidId} | セット応募詳細取得 |
| agreeSetBid | POST | /api/set-bids/{setBidId}/agree | セット一括合意 |

## セキュリティ考慮事項

- 認証: 必要
- 認可: 対象セット応募が参照する全案件の配送依頼企業（自テナント）のみ
- CSRF 対策: JWT Bearer 方式
- その他: なし
