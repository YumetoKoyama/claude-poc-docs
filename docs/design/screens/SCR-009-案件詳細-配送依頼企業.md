# 画面設計: SCR-009 案件詳細画面（配送依頼企業）

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 画面概要

- 画面 ID: SCR-009
- URL パス: `/jobs/{jobId}`（配送依頼企業）
- 対応ユースケース: UC-008, UC-009（編集・削除）, UC-014（連絡）, UC-015（最終条件提示）, UC-016（合意する）, UC-019（完了確認）
- 関連機能要件: `docs/requirements/functional/交渉合意成約.md`, `docs/requirements/functional/運送ステータス報告.md`, `docs/requirements/functional/案件削除.md`, `docs/requirements/functional/案件登録.md`
- 所属レイアウト: 共通シェル

## 入力

| 項目名 | 型 | 必須 | 送信先（operationId.フィールド） | 説明 |
|-------|----|------|--------------------------|------|
| 案件編集内容 | 案件登録.md の入力項目に準ずる | 編集時必須 | `updateJob.*` | 募集中・交渉中のみ活性 |
| 削除確認操作 | ボタン | — | `deleteJob`（パラメータなし） | 募集中・交渉中のみ活性 |
| メッセージ本文 | 文字列 | 送信時必須 | `createMessage.body`（対象は選択中の応募 `bidId`） | |
| 最終条件（金額・場所日時・トラック種別） | 各項目 | 提示時必須 | `proposeFinalOffer.*` | |
| 合意操作 | ボタン | — | `agreeFinalOffer`（単体応募）/ `agreeSetBid`（セット応募） | |
| 破棄操作 | ボタン | — | `discardFinalOffer`（パラメータなし） | 提示中の最終条件提示に対して合意せず破棄する（Q-J2、再提示不可・合意 or 破棄のみ選択可）。単体応募のみ（セット応募は個別破棄不可、連動確認モーダル参照） |
| 完了確認操作 | ボタン | — | `confirmCompletion`（パラメータなし） | |

## 出力・表示内容

| 項目名 | 型 | データ源（operationId.フィールド） | 説明 |
|-------|----|--------------------------|------|
| 案件基本情報 | オブジェクト | `getJobById.*`（fromLocation/fromDatetime/toLocation/toDatetime/volumeM3/cargoType/truckType/attributes/desiredAmount/remarks） | |
| 案件ステータス | 文字列 | `getJobById.status` | |
| 応募一覧（会社名・金額・連絡先） | リスト | `listBidsForJob.items[]`（carrierCompanyName, carrierCompanyPhone, carrierCompanyEmail, amount, status） | 配送依頼企業視点は全応募の実名・金額・連絡先（BR-020）を表示 |
| セット応募の連動案件表示 | リスト | `getSetBidById.bids[]` | 対象応募がセット応募の場合、応募の setBidId をキーに取得 |
| 連絡履歴 | リスト | `listMessages.items[]`（senderDisplayName, body, createdAt） | 各メッセージに送信者ユーザー名を表示 |
| 現在の最終条件提示状況 | オブジェクト | `getBidById.currentFinalOffer` | |
| 削除済み表示 | 真偽値 | `getJobById.logicallyDeleted` | |
| 成約スナップショット | オブジェクト | `getJobById.contract`（成約済以降のみ非null） | 合意金額・場所日時・トラック種別・双方企業/担当者情報・合意時点メッセージ |
| 運送ステータス | 日時群 | `getJobById.departureReportedAt` / `completionReportedAt` / `completionConfirmedAt` | |

## バリデーションメッセージ

| 項目名 | 条件 | メッセージ |
|-------|------|-----------|
| 案件編集内容 | 成約済以降のステータス | MSG-021: 現在のステータスでは更新・削除できません。 |
| 削除操作 | 成約済以降のステータス | MSG-021（導線自体を非表示） |
| 削除確認 | 削除確認ダイアログ | MSG-012: この案件を削除します。よろしいですか？（応募していた運送会社に通知されます） |
| メッセージ送信 | 成約済以降 | MSG-021 |
| 最終条件提示 | 入力検証エラー | MSG-008: 最終条件の入力に誤りがあります。内容を確認してください。 |
| 合意操作 | 相手の状態変化（他応募先行成約, Q-J3） | MSG-009: 相手の状態が変化したため、この操作は完了できません。最新の状態を確認してください。 |
| 完了確認 | 完了報告未実施 | MSG-011: 運送会社の完了報告が完了していないため、完了確認はできません。 |

## 画面遷移（この画面から）

| トリガー | 遷移先（画面 ID） | 条件 |
|---------|--------------------|------|
| セット応募の応募一覧クリック | SCR-009-01 | 対象がセット応募（`bids.setBidId` 非null） |
| 完了確認後「評価する」 | SCR-010 | `getJobById.status == COMPLETED` |
| 一覧へ戻る | SCR-008 | — |
| 削除成功 | SCR-008 | `deleteJob` が 204 |

## セキュリティ考慮事項

- 認証: 必要
- 認可: 自テナントが当事者の案件のみ（他テナントの案件は 404、セキュリティ設計.md 5節 認可設計）
- CSRF 対策: JWT Bearer 方式
- その他: 個人担当者連絡先（`requesterContactName`/`Phone` 相当の相手側情報）は成約済以降のみ非null（BR-020、サービス層でマスク）
