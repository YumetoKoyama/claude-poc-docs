# 画面設計: SCR-007 案件登録画面

> ID 凡例: [docs/凡例.md](../../凡例.md) 参照

## 画面概要

- 画面 ID: SCR-007
- URL パス: `/jobs/new`
- 対応ユースケース: UC-007（案件登録）
- 関連機能要件: `docs/requirements/functional/案件登録.md`
- 所属レイアウト: 共通シェル

## 入力

| 項目名 | 型 | 必須 | 送信先（operationId.フィールド） | 説明 |
|-------|----|------|--------------------------|------|
| from 場所 | 文字列 | 必須 | `createJob.fromLocation` | |
| from 時刻 | 日時 | 必須 | `createJob.fromDatetime` | |
| to 場所 | 文字列 | 必須 | `createJob.toLocation` | |
| to 時刻 | 日時 | 必須 | `createJob.toDatetime` | |
| 容積 | 数値 | 必須 | `createJob.volumeM3` | 単位 m3（Q-DM3） |
| 属性 | 複数選択 | 任意 | `createJob.attributes[]` | 危険物/要冷蔵/要冷凍/割れ物 |
| 物品種別 | 単一選択 | 必須 | `createJob.cargoType` | |
| 希望トラック種別 | 単一選択 | 必須 | `createJob.truckType` | |
| 希望金額 | 数値 | 必須 | `createJob.desiredAmount` | 単位: 円（税別）、整数 |
| 備考 | 文字列 | 任意 | `createJob.remarks` | |

## 出力・表示内容

| 項目名 | 型 | データ源（operationId.フィールド） | 説明 |
|-------|----|--------------------------|------|
| 登録結果 | 案件ID・ステータス | `createJob.jobId` / `createJob.status` | 成功時 SCR-008 へ遷移 |

## バリデーションメッセージ

| 項目名 | 条件 | メッセージ |
|-------|------|-----------|
| 各入力項目 | 必須未入力、または from/to 日時不整合 | MSG-001: 必須項目が未入力、またはfrom/toの日時に不整合があります。内容を確認してください。 |

## 画面遷移（この画面から）

| トリガー | 遷移先（画面 ID） | 条件 |
|---------|--------------------|------|
| 登録成功 | SCR-008 | `createJob` が 201 |
| キャンセル | SCR-006 | — |

## API 対応

| operationId | HTTP メソッド | パス | 概要 |
|-------------|-------------|------|------|
| createJob | POST | /api/jobs | 案件登録 |

## セキュリティ考慮事項

- 認証: 必要
- 認可: 配送依頼企業ユーザーのみ（運送会社は 403、セキュリティ設計.md 5節 認可設計）
- CSRF 対策: JWT Bearer 方式
- その他: `fromLocation`/`toLocation`/`remarks` は XSS 対策として `pattern: '^[^<>]*$'` を適用（HTML タグ不可）

## 受け入れ条件

| AC-ID | 区分 | Given（前提状態） | When（操作） | Then（期待結果） | 関連 BR | operationId |
|-------|------|-----------------|------------|----------------|--------|-------------|
| AC-001 | 正常系 | 案件登録項目をすべて有効な値で入力した状態 | 登録操作を行う | 案件がステータス「募集中」で作成され一覧に表示される | BR-011 | createJob |
| AC-101 | 異常系 | 必須項目が未入力、またはfrom時刻がto時刻より後になっている等の不整合がある状態 | 登録操作を行う | MSG-001が表示され登録は行われない | — | createJob |
