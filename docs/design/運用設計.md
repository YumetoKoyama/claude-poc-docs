# 運用設計

> ID 凡例: [docs/凡例.md](../凡例.md) 参照
> 要件 `非機能要件.md`「運用・障害対応」節の運用・可用性要求値を設計へ落とし込む。

## 1. 監視

| 項目 | 内容 |
|------|------|
| 死活監視 | BE アプリケーションのヘルスチェックエンドポイント（Spring Boot Actuator `/actuator/health` 相当）を定期監視する（Q-NF8）。監視間隔・具体的な監視ツールはインフラ構築フェーズで確定 |
| 監視対象 | BE アプリケーションプロセス、DB 接続可用性、夜間バッチ（EXT-010）の実行結果 |
| ログ監視 | ERROR レベルログ（`共通部品設計.md` 4節）の継続発生を監視対象とする。メール送信失敗（EXT-001）・バッチ失敗（EXT-010）の継続検知を含む（Q-NF8） |

## 2. アラート

| 項目 | 内容 |
|------|------|
| 検知条件 | ヘルスチェック失敗、メール送信の継続失敗、バッチの継続失敗（複数日） |
| 通知先 | 運用担当者（メール、Q-NF8） |
| 通知内容 | 障害種別・発生時刻・影響範囲の概要（詳細ログは対象外、機微情報を含めない） |

## 3. バックアップ・リストア

| 項目 | 内容 |
|------|------|
| 対象データ | DB 全データ（PostgreSQL 16） |
| 頻度 | 日次（1 日 1 回、Q-NF7） |
| 世代数 | 7 世代（1 週間分）保持（Q-NF7） |
| バックアップ方式 | PostgreSQL の論理バックアップ（`pg_dump`）またはマネージド DB のスナップショット機能（具体方式はインフラ構築フェーズで確定） |
| リストア手順 | (1) 対象世代のバックアップを取得 (2) 新規/既存 DB インスタンスへリストア (3) BE アプリケーションの DB 接続先を切り替え (4) ヘルスチェックで復旧確認。デモ用途のため RTO/RPO の目標値は明示しない（`非機能要件.md` 可用性節） |

## 4. ジョブ運用

| 項目 | 内容 |
|------|------|
| 対象ジョブ | 案件物理削除バッチ（EXT-010、`バッチ設計.md` 参照） |
| 実行時間枠 | 深夜 2:00〜4:00（Q-NF9） |
| 実行結果確認 | アプリケーションログで成功/失敗件数を確認する。継続失敗時は監視アラート（本書 2節）で検知する |
| 手動再実行 | 障害復旧後、対象条件（`deleted_at <= now() - 24h`）を満たす案件は次回定時実行で自動的に再試行されるため、手動再実行の手順は必須としない（第 1 版） |

## 5. ログ保持・監査ログ出力箇所

| 項目 | 内容 |
|------|------|
| ログ保管期間 | 90 日間（Q-NF10） |
| 監査対象操作 | 詳細な操作ログの個別確認機能は第 1 版では持たない（要件`概要.md`対象外）。アプリケーションログ（`共通部品設計.md` 4節の INFO レベル）に業務操作の開始/終了を記録するに留める |
| 監査ログ出力箇所 | 認証（ログイン成功/失敗、`auth/`）、成約処理（`deal/`）、案件削除（`job/`）、評価登録（`delivery/`）の各 UseCase 実行時にアプリケーションログへ記録する。個人情報（氏名・電話番号・メールアドレス）はログに出力しない（R-SEC-050） |
| 通知データの保持 | 90 日間（Q-NF10）。通知物理削除バッチ（EXT-011、`バッチ設計.md` 3節）が毎夜 `idx_notifications_created_at` を用いて作成から 90 日経過した通知を削除する |

## 6. 稼働時間帯・計画停止

| 項目 | 内容 |
|------|------|
| 稼働時間帯 | 24 時間稼働を目標（Q-NF9）。目標稼働率・RTO/RPO はデモ用途のため明示しない |
| 計画停止 | 実施を許容する。周知は「工事中」画面（SCR-019）および通知機能での事前案内（Q-NF9） |
| 計画停止の実施方法 | デプロイ時等の計画停止は、事前に SCR-019 の案内文言を更新し、停止前に通知（`notification/` モジュール）で利用者へ周知する運用とする（具体的な自動化フローは第 1 版では設計しない） |

## 7. 問い合わせ対応

| 項目 | 内容 |
|------|------|
| 受付方法 | 「工事中」画面（SCR-019）での案内（本格的な問い合わせフォームは次期対応） |
| 運用フロー | 第 1 版では自動チケット化等は行わず、SCR-019 に記載された案内（連絡先等）を通じたオフライン対応とする |
