# 配送マッチング BtoB Design System

配送を依頼したい企業（配送依頼企業 `REQUESTER`）と、荷物を運ぶ運送会社（`CARRIER`）を結ぶ配送マッチング Web アプリ（第1版）のデザインシステム。案件登録 → 応募 → 交渉 → 成約 → 運送 → 完了 → 評価までの業務プロセスを扱う、**デスクトップ専用（1280px以上、レスポンシブ非対応）の BtoB 業務システム**。

参照元: 添付 `_共通.md`（UI設計ブリーフ）。カラー・タイポグラフィ・スペーシング等の詳細トークンは未指定だったため、本システムが「信頼感・正確性を重視した実務的な業務システム」という方針に沿って初期提案したもの。画面別の生成は次のステップ。

## コンテンツ・ファンダメンタルズ

- 言語: 日本語固定
- 敬体（です・ます調）。利用者を非難しない客観的表現（「間違えています」ではなく「入力に誤りがあります」）
- ボタン等のラベルは体言止め（「新規登録」「案件登録」「削除」「キャンセル」「合意する」）
- 絵文字は使用しない（信頼感・正確性を重視する業務トーンと整合しないため）
- マイクロコピーはプロフェッショナル・簡潔。親しみやすさより客観性・信頼性を優先

## ビジュアル・ファンダメンタルズ

- **配色**: ニュートラルグレー基調（寒色寄り、oklch chroma 極小）＋アクセント1色（トラストブルー、oklch hue 250）。ステータスバッジのみ success/warning/danger/violet の意味色を併用し、通常のUI操作色はアクセント1色に限定する。
- **タイポグラフィ**: Noto Sans JP（和文）+ Inter（欧字・数字の字形統一）。本文基準14px、密度優先の業務システムスケール。見出しも控えめなサイズ差（22px/28px止まり）で華美にしない。
- **スペーシング**: 4px グリッド（4/8/12/16/24/32/48/64）。
- **角丸**: 4px〜8px の控えめな角丸のみ。ピル形状は StatusBadge/Tag 等の小要素に限定。
- **シャドウ**: 低エレベーション（sm/md/lg の3段階、いずれも淡い）。過度な立体感・グラデーションは使わない。
- **境界線**: カード・パネルはボーダー＋わずかな影で区切る。左ボーダーのアクセントカラー装飾は使わない。
- **ホバー/プレス**: ボタンは背景色をわずかに濃く（primary hover は accent-600）。ゴースト/セカンダリはニュートラル背景を薄く敷く。押下時のスケール変化は使わない（実務ツールとして過度なモーションを避ける）。
- **アニメーション**: 未規定。フェード等の過度な演出は避け、状態変化は瞬時 or ごく短いトランジション（~120ms）に留める。
- **画像/イラスト**: 本システムに写真・イラスト要素はなし（業務データ中心のUI）。

## アイコノグラフィ

- アイコンライブラリの指定なし。本システムでは Feather Icons 相当のシンプルな線画アイコン（stroke-width 2）をインラインSVGで最小限使用（通知ベル、ナビゲーションアイコン、モーダル閉じるボタン等）。
- 絵文字は使用しない。

## 状態規約（States）

| 状態 | 表示 |
|---|---|
| 空データ | 一覧領域中央に「該当データなし」（Table の emptyMessage） |
| バリデーションエラー（VALIDATION_ERROR） | Field 直下にインライン表示 |
| その他エラー（FORBIDDEN/NOT_FOUND/CONFLICT/INTERNAL_ERROR等） | 画面右上 Toast 表示 |
| 権限不足（403） | Toast に含む。テナント越境は404（非表示） |

## レスポンシブ / アクセシビリティ方針

- デスクトップ専用固定レイアウト（最小幅 1280px）。ブレークポイントなし。
- WCAG準拠・キーボード操作・スクリーンリーダー対応は第1版の必須要件外（将来対応）。一般的な可読コントラストのみ確保。

## 未確定事項・要確認

- HEX値・具体トーンは初期提案であり未最終承認（`_共通.md` 10節参照）。
- Loading状態の表示仕様、権限不足時の正確なコピー文言は未定義。
- ボタン命名規約・コンポーネント命名規約の明文化は未実施。

## ファイル構成

- `styles.css` — グローバルCSSエントリポイント（`tokens/*.css` を import）
- `tokens/` — colors / typography / spacing / radius / shadow / layout / base
- `components/layout/` — AppHeader, GlobalNav, Card, Panel
- `components/forms/` — Button, Field
- `components/feedback/` — Modal, Toast, StatusBadge, Tag
- `components/data/` — Table, Pagination
- `guidelines/` — カラー・タイポグラフィ・スペーシング・エレベーションの見本カード
- `SKILL.md` — Claude Code 等での利用向けスキル定義
