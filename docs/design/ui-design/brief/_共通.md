## 1. プロダクト概要

配送を依頼したい企業（配送依頼企業）と、実際に荷物を運ぶ運送会社とを結び付ける「配送マッチング Web アプリ」の第 1 版。案件（積荷）の登録から、運送会社による応募、金額・条件交渉、合意・成約、運送実施、完了確認、相互評価までの一連の業務プロセスを Web 上で完結させる BtoB 業務システムである。

決済機能・成約後キャンセル・ロール分離承認フロー・二要素認証・リアルタイム Push 通知は第 1 版のスコープ外。第 1 版はデスクトップ PC ブラウザ専用（レスポンシブ対応なし）。

- 主要ユーザー種別（ロール）: 配送依頼企業ユーザー（`REQUESTER`）、運送会社ユーザー（`CARRIER`）
- 参照: `docs/design/概要.md` 1節、`docs/requirements/概要.md` 1〜3節

## 3. トーン & ボイス

- 言語: 日本語固定（第 1 版は多言語対応なし）
- 敬体 / 常体: 敬体（です・ます調）で統一し、利用者を非難しない客観的な表現とする（例:「間違えています」ではなく「入力に誤りがあります」）
- 絵文字使用可否: 明記なし（未確定事項に記載。信頼感・正確性を重視する業務システムのトーンとは整合しない可能性がある点に留意）
- ボタン命名規則: 明記なし。ただし各画面設計 md のトリガー欄では体言止めのラベルが一貫して使われている（例:「新規登録」「案件登録」「削除」「キャンセル」「合意する」）
- マイクロコピー方針: プロフェッショナル・簡潔。親しみやすさよりも客観性・信頼性を優先する
- 参照: `docs/requirements/ブランドガイドライン.md` 1節「ブランド方針」、`docs/requirements/メッセージ一覧.md` 冒頭「文言トーン方針（Q-MSG1）」

## 5. 共通コンポーネント

各コンポーネントの寸法・詳細トークンは Design System が未確定のため Claude Design の既定に従う。振る舞い・構成要素は設計書から以下のとおり定まっている。画面別ブリーフではこれを参照するのみとし、再掲しない。

- **AppHeader**: ロゴ／テナント名／ユーザー名／通知ベル（未読件数バッジ、9件超は「9+」表示・BR-024）／ユーザーメニュー（ログアウト）。ロゴクリックでロール別ダッシュボード（配送依頼企業: SCR-006 / 運送会社: SCR-013）へ遷移。
- **GlobalNav（SideNav 相当）**: ダッシュボード／案件系メニュー（`tenantType` で出し分け。配送依頼企業: 案件登録・案件一覧 / 運送会社: 募集中一覧・交渉中一覧）／取引履歴／ユーザー追加。
- **Footer**: 「お問い合わせ」導線（SCR-019 工事中画面へ遷移）。
- **Button**: variant（primary/secondary 等）・状態は未指定。ラベルは体言止め規約に従う（3節参照）。
- **Field**: ラベル・必須印・エラー表示の詳細トークンは未指定。バリデーションエラーはフィールド直下表示（6節参照）。
- **Modal**: SCR-009-01（セット応募連動確認モーダル）・SCR-015-01（セット応募選択モーダル）が該当。独立レイアウトを持たず、親画面（SCR-009 / SCR-014・SCR-015）上のオーバーレイとして共通シェルの一部で表示する。
- **Table / List**: 案件一覧・応募一覧・通知一覧・取引履歴一覧等で使用。ページング情報（`pageMeta`）を伴う一覧は Pagination を併設する。
- **StatusBadge**: 案件ステータス（ST-001 募集中／ST-002 交渉中／ST-003 成約済／ST-004 運送中／ST-005 完了／ST-006 評価済。ST-007 キャンセルは第1版未使用）、応募ステータス（ST-101 応募中／ST-102 成約／ST-103 クローズ（他社成約）／ST-104 クローズ（セット連動不成立））をバッジ表示する。配色マッピングは未指定。
- **Tag**: 「new」表示（24時間以内更新、SC-002）、「削除済み」表示（論理削除、案件削除.md AC-201）に使用。
- **Toast / インライン エラー**: `ErrorResponse` を受けた API 呼び出しは `code=VALIDATION_ERROR` の場合フィールド直下、それ以外（`FORBIDDEN`/`NOT_FOUND`/`CONFLICT`/`INTERNAL_ERROR` 等）は画面右上のトースト表示。
- **Pagination**: `listJobs`/`listMyBids`/`listNotifications` 等の `pageMeta` に対応。
- 参照: `docs/design/screens/共通レイアウト.md`、`docs/design/screens/*.md` 各「出力・表示内容」「バリデーションメッセージ」、`docs/requirements/データモデル.md`（ST-001〜ST-007, ST-101〜ST-104）

## 6. 状態表現規約（States）

| 状態 | 表示 | 既定コピー |
|------|------|---------|
| 空データ | 一覧領域にメッセージ表示 | 該当データなし（例: 取引履歴/AC-101「取引履歴が1件も存在しない状態」の表示文言。正確な文言は未指定） |
| Loading | 未指定（設計書に記載なし） | 未指定 |
| エラー（VALIDATION_ERROR） | フィールド直下にインライン表示 | 各画面のバリデーションメッセージ（MSG-XXX）に従う |
| エラー（それ以外: FORBIDDEN/NOT_FOUND/CONFLICT/INTERNAL_ERROR 等） | 画面右上トースト表示 | `ErrorResponse.message` に従う |
| 権限不足（403） | 上記トーストに含まれる。テナント越境は 404（コンテンツ非表示） | 未指定（正確な文言は未確定） |

- 参照: `docs/design/screens/共通レイアウト.md`「共通の挙動」、`docs/design/概要.md` 2節（テナント越境404／権限不足403）、`docs/requirements/取引履歴.md` AC-101

## 7. レスポンシブ方針

- 想定デバイス優先度: デスクトップ必須（1280px 以上）。タブレット・モバイルは対象外（表示崩れの保証なし）
- ブレークポイント: なし（第1版はレスポンシブ戦略を採用しない、デスクトップ専用固定レイアウト）
- 対象ブラウザ: Chrome/Edge/Safari の直近2メジャーバージョン
- 参照: `docs/requirements/ブランドガイドライン.md` 3節「想定デバイスとレスポンシブ方針」
