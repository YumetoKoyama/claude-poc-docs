---

## 1. Goal（この画面の目的）

- 一次目的: 案件の詳細確認、応募一覧の審査、連絡・交渉、最終条件提示・合意、完了確認、削除までを一画面で完結させる
- 参照: `docs/design/screens/SCR-009-案件詳細-配送依頼企業.md`「画面概要」

## 2. Audience（差分）

- ロール: 配送依頼企業ユーザーのみ（自テナントが当事者の案件のみ、他テナントは404）

## 3. Content（表示する情報・固有コピー）

| 領域 | 表示要素 | 出典 |
|------|---------|------|
| 案件基本情報 | from/to 場所・時刻、容積、物品種別、トラック種別、属性、希望金額、備考、ステータス | 設計 md「出力・表示内容」 |
| 応募一覧 | 会社名・金額・連絡先（配送依頼企業視点は全応募の実名・金額・連絡先を表示、BR-020） | 設計 md「出力・表示内容」 |
| セット応募連動案件 | 対象応募がセット応募の場合、連動案件一覧をリンク表示（SCR-009-01 起動） | 設計 md「出力・表示内容」 |
| 連絡履歴 | メッセージ本文＋送信者ユーザー名＋日時 | 設計 md「出力・表示内容」 |
| 最終条件提示状況 | 現在提示中の最終条件（金額・場所日時・トラック種別） | 設計 md「出力・表示内容」 |
| 成約スナップショット | 合意金額・場所日時・トラック種別・双方企業/担当者情報・合意時点メッセージ（成約済以降のみ非null） | 設計 md「出力・表示内容」 |
| 運送ステータス | 運送開始報告日時・完了報告日時・完了確認日時 | 設計 md「出力・表示内容」 |
| 削除済み表示 | 論理削除済みタグ | 設計 md「出力・表示内容」 |
| 画面固有のエラーメッセージ | MSG-021（成約済以降の操作拒否）、MSG-012（削除確認ダイアログ）、MSG-008（最終条件入力エラー）、MSG-009（相手状態変化）、MSG-011（完了報告未実施） | 設計 md「バリデーションメッセージ」 |

- 参照: `docs/design/screens/SCR-009-案件詳細-配送依頼企業.md`

## 4. Layout（領域分割と情報階層）

- 主要レイアウト: 案件基本情報（上部）→ 応募一覧/交渉パネル（中央）→ 連絡履歴（下部 or サイドパネル）
- ステータス（募集中/交渉中/成約済/運送中/完了/評価済）に応じて編集・削除・完了確認等のボタン活性状態が変化する（StatusBadge と連動）
- 参照: 設計 md「画面概要」

## 5. States（差分）

| 状態 | 振る舞い | 表示要素 |
|------|---------|--------|
| 成約済以降のステータス | 編集・削除・メッセージ送信・最終条件提示の導線を非表示/非活性化 | MSG-021 |
| 削除確認 | 確認ダイアログ表示 | MSG-012: この案件を削除します。よろしいですか？（応募していた運送会社に通知されます） |
| 相手の状態変化（他応募先行成約、Q-J3） | 合意操作が完了できない | MSG-009 |
| 完了報告未実施 | 完了確認ボタンが非活性 | MSG-011 |
| セット応募の応募一覧クリック | SCR-009-01 モーダルを起動 | — |

- 参照: 設計 md「バリデーションメッセージ」

## 6. Interaction（操作と遷移）

- セット応募の応募一覧クリック → SCR-009-01（モーダル起動）
- 完了確認後「評価する」→ SCR-010（`status == COMPLETED`）
- 一覧へ戻る → SCR-008、削除成功（204）→ SCR-008
- 破棄操作（`discardFinalOffer`）は単体応募のみ可能（セット応募は個別破棄不可）
- 参照: 設計 md「画面遷移（この画面から）」

## 7. 画面固有のバリデーション

| フィールド | 制約 | エラーメッセージ |
|---------|------|--------------|
| 案件編集内容 | 成約済以降のステータス | MSG-021: 現在のステータスでは更新・削除できません。 |
| 削除確認 | 確認ダイアログ | MSG-012: この案件を削除します。よろしいですか？（応募していた運送会社に通知されます） |
| 最終条件提示 | 入力検証エラー | MSG-008: 最終条件の入力に誤りがあります。内容を確認してください。 |
| 合意操作 | 相手の状態変化（Q-J3） | MSG-009: 相手の状態が変化したため、この操作は完了できません。最新の状態を確認してください。 |
| 完了確認 | 完了報告未実施 | MSG-011: 運送会社の完了報告が完了していないため、完了確認はできません。 |

- 参照: 設計 md「バリデーションメッセージ」

## 8. 画面遷移（この画面から）

| 操作 | 遷移先画面 ID | 備考 |

| 一覧へ戻る | SCR-008 | — |

- 参照: `docs/design/screens/画面遷移.md`

## 9. 強調したい点（任意）

- 個人担当者連絡先（相手側情報）は成約済以降のみ表示される（BR-020）。UI 上、成約前は「連絡先は成約後に表示されます」等の状態が伝わる表現が望ましい。
- ステータス（募集中→交渉中→成約済→運送中→完了→評価済）に応じた操作可否の切り替えが分かりやすいこと。
