- 対応する設計 md: `docs/design/screens/SCR-015-案件詳細-運送会社.md`
- 対応する機能要件: `docs/requirements/functional/応募.md`、`docs/requirements/functional/交渉合意成約.md`、`docs/requirements/functional/運送ステータス報告.md`
- 共通ブリーフ: `docs/design/ui-design/brief/_共通.md`

> このファイルは **共通ブリーフからの差分のみ** を記述する。SCR-009（配送依頼企業向け案件詳細）の運送会社視点版。機能が多い画面。

---

## 1. Goal（この画面の目的）

- 一次目的: 案件への応募・応募編集・連絡交渉・最終条件提示/合意・運送開始報告・完了報告までを一画面で完結させる
- 参照: `docs/design/screens/SCR-015-案件詳細-運送会社.md`「画面概要」

## 2. Audience（差分）

- ロール: 運送会社ユーザーのみ。募集中案件は全運送会社に公開、交渉中以降は自社が応募した案件のみアクセス可（それ以外404）

## 3. Content（表示する情報・固有コピー）

| 領域 | 表示要素 | 出典 |
|------|---------|------|
| 案件基本情報 | from/to 場所・時刻、容積、物品種別、トラック種別、属性、希望金額、備考 | 設計 md「出力・表示内容」 |
| 配送依頼企業情報 | 会社名・連絡先（募集中・交渉中でも表示可、BR-020）、個人担当者連絡先（成約済以降のみ非null） | 設計 md「出力・表示内容」 |
| 自社応募状況 | 応募内容（未応募ならnull、応募ボタン表示） | 設計 md「出力・表示内容」 |
| 競合表示 | 他社金額（匿名ラベル・金額null、成約後は競合情報自体を返さない、BR-021） | 設計 md「出力・表示内容」 |
| 連絡履歴 | メッセージ本文＋送信者ユーザー名＋日時 | 設計 md「出力・表示内容」 |
| 最終条件提示状況 | 現在提示中の最終条件 | 設計 md「出力・表示内容」 |
| 成約スナップショット | 成約済以降のみ非null | 設計 md「出力・表示内容」 |
| 運送ステータス日時 | 運送開始報告日時・完了報告日時 | 設計 md「出力・表示内容」 |
| 画面固有のエラーメッセージ | MSG-004/005（応募上限・締切）、MSG-017（重複応募）、MSG-002（応募編集エラー）、MSG-021（成約済以降）、MSG-008（最終条件入力エラー）、MSG-009（相手状態変化）、MSG-010（運送開始報告不可） | 設計 md「バリデーションメッセージ」 |

- 参照: `docs/design/screens/SCR-015-案件詳細-運送会社.md`

## 4. Layout（領域分割と情報階層）

- 主要レイアウト: 案件基本情報（上部）→ 自社応募/交渉パネル（中央、未応募時は応募フォーム、応募済は応募内容＋交渉UI）→ 連絡履歴（下部 or サイドパネル）
- 参照: 設計 md「画面概要」

## 5. States（差分）

| 状態 | 振る舞い | 表示要素 |
|------|---------|--------|
| 未応募 | 応募フォームを表示 | 応募金額入力欄 |
| 応募済（成約前） | 応募内容編集フォームを表示 | 編集後の応募金額入力欄 |
| 成約済以降 | 応募編集・メッセージ送信・最終条件提示の導線を非表示/非活性化 | MSG-021 |
| ステータスが「成約済」でない | 運送開始報告ボタン非活性 | MSG-010: 現在のステータスでは運送開始報告はできません。 |
| 対象応募が非セット | 合意操作（`agreeFinalOffer`）を表示 | — |

- 参照: 設計 md「バリデーションメッセージ」「入力」

## 6. Interaction（操作と遷移）

- 「セット応募」操作 → SCR-015-01（モーダル起動）
- 完了報告後「評価する」→ SCR-017（`status == COMPLETED` かつ完了報告実施済み）
- 一覧へ戻る → SCR-014 / SCR-016（遷移元に応じる）
- 参照: 設計 md「画面遷移（この画面から）」

## 7. 画面固有のバリデーション

| フィールド | 制約 | エラーメッセージ |
|---------|------|--------------|
| 応募 | 上限・締切超過 | MSG-004 / MSG-005 |
| 応募 | 重複応募 | MSG-017 |
| 応募編集 | 入力検証エラー | MSG-002: 応募内容の入力に誤りがあります。内容を確認してください。 |
| 応募編集 | 成約済以降 | MSG-021 |
| 最終条件提示 | 入力検証エラー | MSG-008 |
| 合意操作 | 相手の状態変化 | MSG-009 |
| 運送開始報告 | ステータスが「成約済」でない | MSG-010 |

- 参照: 設計 md「バリデーションメッセージ」

## 8. 画面遷移（この画面から）

| 操作 | 遷移先画面 ID | 備考 |
|------|------------|------|
| 「セット応募」操作 | SCR-015-01 | モーダル起動 |
| 完了報告後「評価する」 | SCR-017 | `status == COMPLETED` |
| 一覧へ戻る | SCR-014 / SCR-016 | 遷移元に応じる |

- 参照: `docs/design/screens/画面遷移.md`

## 9. 強調したい点（任意）

- 競合表示は金額のみ匿名化されるため、他社の存在は分かるが金額は分からない UI 表現にすること（BR-021）。
- 運送開始・完了報告は取消不可のため、報告操作には確認ステップを設けることが望ましい（設計書は取消不可のみ明記、確認 UI の要否は未確定）。
