## 1. Goal（この画面の目的）

- 一次目的: ログイン ID・パスワードによる認証を行い、テナント種別に応じたダッシュボードへ遷移させる
- 二次目的: 新規登録・パスワード忘失時の導線を提供する
- 参照: `docs/design/screens/SCR-001-ログイン.md`「画面概要」

## 2. Audience（差分）

- ロール: 未認証ユーザー（共通ブリーフに従う）
- 参照: `docs/design/ui-design/brief/_共通.md` 2節

## 3. Content（表示する情報・固有コピー）

| 領域 | 表示要素 | 出典 |
|------|---------|------|
| メイン | ログイン ID 入力、パスワード入力（`type=password`）、ログインボタン | 設計 md「入力」 |
| 補助領域 | 「新規登録」リンク、「パスワードをお忘れですか」リンク | 設計 md「画面遷移」 |
| 画面固有のエラーメッセージ | MSG-015（認証失敗）、MSG-016（連続5回失敗によるロック） | 設計 md「バリデーションメッセージ」 |

- 参照: `docs/design/screens/SCR-001-ログイン.md`

## 4. Layout（領域分割と情報階層）

- 主要レイアウト: 中央寄せの単一カラムフォーム
- ファーストビューに必須の要素: ログイン ID・パスワード入力欄、ログインボタン
- 情報の重要度順序: ①認証フォーム ②補助リンク（新規登録／パスワード忘失）
- レイアウト区分: 認証前独立レイアウト（共通シェルに乗らない）
- 参照: 設計 md「画面概要」、`docs/design/screens/共通レイアウト.md`「レイアウト区分」

## 5. States（差分）

| 状態 | 振る舞い | 表示要素 |
|------|---------|--------|
| 認証失敗 | ID/パスワードいずれの誤りかを区別しない一律表示（ユーザー列挙防止） | MSG-015 |
| ロック中（連続5回失敗、Q-NF2） | 15分間ログイン操作を拒否 | MSG-016 |

- 参照: 設計 md「バリデーションメッセージ」「セキュリティ考慮事項」

## 6. Interaction（操作と遷移）

- ログイン成功: `tenantType == REQUESTER` → SCR-006 / `tenantType == CARRIER` → SCR-013
- 「新規登録」→ SCR-002、「パスワードをお忘れですか」→ SCR-003
- 任意の認証済み画面からのログアウト操作 → 本画面へ遷移
- 参照: 設計 md「画面遷移（この画面から）」

## 7. 画面固有のバリデーション

| フィールド | 制約 | エラーメッセージ |
|---------|------|--------------|
| ログイン ID / パスワード | 認証失敗 | MSG-015: ログイン ID またはパスワードが正しくありません。 |
| ログイン ID / パスワード | 連続5回失敗（Q-NF2） | MSG-016: ログイン試行回数が上限に達したため、しばらくログインできません。 |

- 参照: 設計 md「バリデーションメッセージ」

## 8. 画面遷移（この画面から）

| 操作 | 遷移先画面 ID | 備考 |
|------|------------|------|
| ログイン成功（配送依頼企業） | SCR-006 | — |
| ログイン成功（運送会社） | SCR-013 | — |
| 「新規登録」 | SCR-002 | — |
| 「パスワードをお忘れですか」 | SCR-003 | — |

- 参照: `docs/design/screens/画面遷移.md`「認証フロー」

## 9. 強調したい点（任意）

- ログイン失敗時は ID・パスワードどちらが誤りかを区別しないメッセージにすること（ユーザー列挙防止の設計意図）。
