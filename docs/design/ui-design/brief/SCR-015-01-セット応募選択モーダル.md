# UI 設計ブリーフ: SCR-015-01 セット応募選択モーダル（運送会社向け）

- 画面 ID: SCR-015-01
- 対応する設計 md: `docs/design/screens/SCR-015-01-セット応募選択モーダル.md`
- 対応する機能要件: `docs/requirements/functional/応募.md`
- 共通ブリーフ: `docs/design/ui-design/brief/_共通.md`（Modal コンポーネント規約を参照）

> このファイルは **共通ブリーフからの差分のみ** を記述する。独立 URL を持たず SCR-014 / SCR-015 上のオーバーレイとして表示する。

---

## 1. Goal（この画面の目的）

- 一次目的: 同一配送依頼企業の複数案件をまとめて1つのセット応募として登録する
- 参照: `docs/design/screens/SCR-015-01-セット応募選択モーダル.md`「画面概要」

## 2. Audience（差分）

- ロール: 運送会社ユーザーのみ

## 3. Content（表示する情報・固有コピー）

| 領域 | 表示要素 | 出典 |
|------|---------|------|
| メイン | 選択可能案件一覧（起動元案件と同一 requesterTenantId の募集中・交渉中案件のみに画面側で絞り込み） | 設計 md「出力・表示内容」 |
| 入力 | セット対象案件選択（複数選択）、各案件の応募金額 | 設計 md「入力」 |

- 参照: `docs/design/screens/SCR-015-01-セット応募選択モーダル.md`

## 4. Layout（領域分割と情報階層）

- SCR-014 / SCR-015 上のオーバーレイモーダル。選択可能案件一覧（チェックボックス付きリスト）＋各案件の金額入力欄＋登録ボタン＋キャンセルボタン
- 参照: 設計 md「画面概要」

## 5. States（差分）

| 状態 | 振る舞い | 表示要素 |
|------|---------|--------|
| 異なる配送依頼企業の案件混在 | UI上追加できないよう制御 | MSG-003: 異なる配送依頼企業の案件を同じセットに含めることはできません。 |
| いずれかの案件で応募上限・締切・重複 | セット応募登録不可 | MSG-004 / MSG-005 / MSG-017 |

- 参照: 設計 md「バリデーションメッセージ」

## 6. Interaction（操作と遷移）

- セット応募成功（201）→ モーダルを閉じ SCR-015 の案件詳細を再取得
- キャンセル → モーダルを閉じ起動元画面（SCR-014 / SCR-015）に留まる
- 参照: 設計 md「画面遷移（この画面から）」

## 7. 画面固有のバリデーション

| フィールド | 制約 | エラーメッセージ |
|---------|------|--------------|
| セット対象選択 | 異なる配送依頼企業の案件混在 | MSG-003 |
| セット応募登録 | いずれかの案件で応募上限・締切・重複 | MSG-004 / MSG-005 / MSG-017 |

- 参照: 設計 md「バリデーションメッセージ」

## 8. 画面遷移（この画面から）

| 操作 | 遷移先画面 ID | 備考 |
|------|------------|------|
| セット応募成功 | SCR-015 | モーダルを閉じ再取得 |
| キャンセル | SCR-014 / SCR-015 | 起動元画面へ |

- 参照: `docs/design/screens/画面遷移.md`

## 9. 強調したい点（任意）

- 選択候補は起動元案件と同一配送依頼企業のものだけに画面側で事前フィルタされている点を UI 上も分かりやすくすること。
