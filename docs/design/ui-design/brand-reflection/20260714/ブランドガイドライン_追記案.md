現行 4節は全項目「未指定」のドラフトです。以下のとおり Claude Design で採択された具体値への更新を提案します。

### 4.1 主要カラー

出典: `handoff/tokens/colors_and_type.css`、`handoff/tokens/DESIGN_SYSTEM_GUIDE.md`

- 方針: ニュートラルグレー基調（寒色寄り、oklch chroma 極小）＋ アクセント1色（トラストブルー、oklch hue 250）。通常のUI操作色はアクセント1色に限定し、意味色（success/warning/danger/violet）はステータスバッジ専用とする。

| 用途 | 論理名（CSS変数） | 値 |
|---|---|---|
| 背景（ページ） | `--color-bg` | `--gray-50` = oklch(0.985 0.002 250) |
| 背景（キャンバス） | `--color-bg-canvas` | `--gray-100` = oklch(0.967 0.003 250) |
| サーフェス（カード等） | `--color-surface` | `--gray-0` = oklch(1 0 0) |
| 境界線 | `--color-border` / `--color-border-strong` | `--gray-200` / `--gray-300` |
| テキスト（主） | `--color-text-primary` | `--gray-900` = oklch(0.19 0.006 250) |
| テキスト（副） | `--color-text-secondary` | `--gray-600` = oklch(0.47 0.008 250) |
| プライマリ（アクセント） | `--color-primary` / hover / active | `--accent-500`(oklch 0.55 0.14 250) / `--accent-600` / `--accent-700` |
| 危険（削除・エラー） | `--color-danger` | `--danger-500` = oklch(0.58 0.19 25) |
| 成功 | `--color-success` | `--success-500` = oklch(0.6 0.14 150) |
| 警告 | `--color-warning` | `--warning-500` = oklch(0.75 0.15 80) |
| 情報 | `--color-info` | `--accent-500` と同値（意味色 info はアクセントと共用） |
| フォーカスリング | `--color-focus-ring` | `--accent-400` |

グレースケール・アクセントスケールは各50〜900/950の11〜9段階で定義済み（詳細は `colors_and_type.css` 参照）。

### 4.2 タイポグラフィ

出典: `handoff/tokens/colors_and_type.css`、`DESIGN_SYSTEM_GUIDE.md`

- フォントファミリ: `Noto Sans JP`（和文）+ `Inter`（欧文・数字の字形統一）、フォールバック `-apple-system, BlinkMacSystemFont, 'Hiragino Sans', 'Yu Gothic', sans-serif`
- ウェイト: regular(400) / medium(500) / bold(700)
- スケール（密度優先、本文基準14px）:

| トークン | サイズ / 行高 |
|---|---|
| xs | 12px / 16px |
| sm | 13px / 20px |
| base（本文） | 14px / 22px |
| md | 16px / 24px |
| lg | 18px / 26px |
| xl（見出し） | 22px / 30px |
| 2xl（見出し最大） | 28px / 36px |

見出しも22px/28px止まりで華美にしない方針。

### 4.3 スペーシング・角丸・シャドウ・フォーカスリング

出典: `handoff/tokens/spacing.css` / `radius.css` / `shadow.css`

- スペーシング: 4pxグリッド（4/8/12/16/20/24/32/40/48/64/80px = `--space-1`〜`--space-20`）
- 角丸: `--radius-sm`(4px) / `--radius-md`(6px) / `--radius-lg`(8px) / `--radius-xl`(12px) / `--radius-full`(9999px、ピル形状はバッジ・タグ等の小要素に限定)
- シャドウ: 3段階、いずれも淡色（`--shadow-sm/md/lg`、`rgba(15,23,42, 0.06〜0.12)`）。過度な立体感・グラデーションは使わない
- フォーカスリング: `0 0 0 3px var(--accent-100), 0 0 0 1px var(--accent-500)`

### 4.4 アイコン

出典: `DESIGN_SYSTEM_GUIDE.md`

- ライブラリ指定なし。Feather Icons相当のシンプルな線画アイコン（stroke-width 2）をインラインSVGで最小限使用（通知ベル・ナビゲーション・モーダル閉じるボタン等）
- 絵文字は使用しない（信頼感・正確性重視のトーンと不整合のため）

## 9. アニメーション / モーション（新設・追記提案）

出典: `DESIGN_SYSTEM_GUIDE.md`

- 既定: 過度な演出を避け、状態変化は瞬時 or ごく短いトランジション（約120ms）に留める
- ボタンのホバー: 背景色をわずかに濃く（primary hoverは `--accent-600`）。ゴースト/セカンダリはニュートラル背景を薄く敷く
- 押下時のスケール変化は使わない（実務ツールとして過度なモーションを避ける方針）

## 抽出不能・人間の確認が必要な項目

- **Loading状態の表示仕様**: handoff から抽出不能（`DESIGN_SYSTEM_GUIDE.md`「未確定事項」に明記あり）。人間の確認・追加のUI検討が必要。
- **権限不足時の正確なコピー文言**: handoff から抽出不能。`メッセージ一覧.md`（Q-MSG1）側での定義が必要。
- **ボタン命名規約・コンポーネント命名規約の明文化**: handoff のコード上は暗黙の命名（`Button`, `Field` 等）のみで、規約文書としての明文化はされていない。UI実装時に確定が必要。
- **ロール別（SHIPPER/CARRIER）のテーマ差**: handoff 上で色分け等のロール別テーマ差は確認できず（両ロールとも同一デザインシステムを共用）。意図的な統一方針か、単に未検討かは人間の確認が必要。
