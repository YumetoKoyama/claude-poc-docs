# SCR-015 案件詳細画面（運送会社）

## 要約

運送会社が応募・応募編集、連絡・最終条件提示・合意、運送開始/完了報告までを行う中心画面。募集中案件は全運送会社に公開、交渉中以降は自社が応募した案件のみアクセス可。

対応ユースケース: UC-011, UC-012, UC-013（応募/セット応募/応募編集）, UC-014（連絡）, UC-015（最終条件提示）, UC-016（合意する）, UC-017（運送開始報告）, UC-018（完了報告） / URL: `/jobs/{jobId}`（運送会社）

---

## 画面遷移（入口と出口）

| 方向 | 画面 | 条件 |
|------|------|------|
| ← 入口 | SCR-014 募集中案件一覧 | 案件行クリック |
| ← 入口 | SCR-016 交渉中・済一覧 | 応募行クリック |
| ← 入口 | SCR-015-01 セット応募選択モーダル | セット応募成功／キャンセル |
| ← 入口 | SCR-017 評価登録 | 登録成功後 |
| ← 入口 | SCR-018 取引履歴一覧 | 案件行クリック（読み取り専用） |
| ← 入口 | SCR-012 通知一覧 | 通知クリック（運送会社） |
| → 出口 | SCR-015-01 セット応募選択モーダル | 「セット応募」操作 |
| → 出口 | SCR-017 評価登録 | 完了報告後「評価する」（`status == COMPLETED` かつ完了報告実施済み） |
| → 出口 | SCR-014 / SCR-016 | 一覧へ戻る（遷移元に応じる） |

## 表示項目

### 入力

| 項目名 | 型 | 必須 | 送信先（operationId.フィールド） | 説明 |
|-------|----|------|--------------------------|------|
| 応募金額 | 数値 | 応募時必須 | `createBid.amount` | 未応募の場合のみ表示 |
| 編集後の応募金額 | 数値 | 編集時必須 | `updateBid.amount` | 自社の応募が成約前の場合のみ活性 |
| メッセージ本文 | 文字列 | 送信時必須 | `createMessage.body`（`bidId`＝自社応募） | |
| 最終条件（金額・場所日時・トラック種別） | 各項目 | 提示時必須 | `proposeFinalOffer.*` | |
| 合意操作 | ボタン | — | `agreeFinalOffer` | 対象応募が非セットの場合のみ |
| 破棄操作 | ボタン | — | `discardFinalOffer` | 提示中の最終条件提示を合意せず破棄（Q-J2） |
| 運送開始報告操作 | ボタン | — | `reportDeparture` | |
| 完了報告操作 | ボタン | — | `reportCompletion` | |

### 出力・表示内容

| 項目名 | 型 | データ源 | 説明 |
|-------|----|--------|------|
| 案件基本情報 | オブジェクト | `getJobById.*` | |
| 配送依頼企業の会社名・連絡先 | 文字列 | `getJobById.requesterCompanyName` 等 | 募集中・交渉中でも表示可（BR-020） |
| 配送依頼企業の個人担当者連絡先 | 文字列（nullable） | `getJobById.requesterContactName` 等 | 成約済以降のみ非null（BR-020） |
| 自社の応募状況 | オブジェクト（nullable） | `getJobById.myBidId` → `getBidById.*` | 未応募なら null（応募ボタン表示） |
| 競合表示（他社金額） | リスト | `listBidsForJob.items[]` | 他社は会社名が匿名ラベル・金額 null。成約後は競合情報自体を返さない（BR-021） |
| 連絡履歴 | リスト | `listMessages.items[]` | 送信者ユーザー名を表示 |
| 成約スナップショット | オブジェクト（nullable） | `getJobById.contract` | 成約済以降のみ非null |
| 運送ステータス日時 | 日時群 | `getJobById.departureReportedAt` / `completionReportedAt` | |

> **⚠️ 制約・注意事項**: 競合表示は金額のみ匿名化しサービス層でフィールドマスクする（BR-021）。運送開始報告は案件が「成約済」でないと MSG-010。応募・応募編集は自社の応募のみ（他は拒否）。

## 受け入れ条件

| AC-ID | 区分 | Given（前提状態） | When（操作） | Then（期待結果） | 関連 BR | operationId |
|-------|------|-----------------|------------|----------------|--------|-------------|
| 運送ステータス報告/AC-001 | 正常系 | 案件ステータスが「成約済」の状態 | 運送開始報告を行う | 案件ステータスが「運送中」に遷移する | BR-017 | reportDeparture |
| 応募/AC-001 | 正常系 | 募集中の案件が応募上限・締切に達していない状態 | 希望金額に同意する、または独自金額を入力して応募する | 応募が登録され初回応募であれば案件ステータスが「交渉中」に遷移する | BR-011 | createBid |
| 運送ステータス報告/AC-101 | 異常系 | 案件ステータスが「成約済」でない状態 | 運送開始報告を行おうとする | 操作導線が表示されず報告できない | BR-017 | reportDeparture |
| 応募/AC-301 | 権限境界 | 自社以外（他テナント）の応募 | 応募内容の編集を試みる | 拒否される（自社の応募のみ編集可能） | — | updateBid |
| 運送ステータス報告/AC-301 | 権限境界 | 自社が当事者ではない案件 | 運送開始報告・完了報告を試みる | 拒否される | — | reportDeparture, reportCompletion |
| 運送ステータス報告/AC-401 | エッジケース | 運送開始・完了報告済みの状態 | 再度取消を試みる | 取消不可（誤操作時は連絡機能で相手と調整） | — | reportDeparture, reportCompletion |

## 対応 API

| operationId | メソッド | パス | 概要 |
|-------------|---------|------|------|
| getJobById | GET | /api/jobs/{jobId} | 案件詳細取得 |
| listBidsForJob | GET | /api/jobs/{jobId}/bids | 応募一覧（競合表示） |
| getBidById | GET | /api/bids/{bidId} | 自社応募詳細 |
| createBid | POST | /api/jobs/{jobId}/bids | 応募 |
| updateBid | PATCH | /api/bids/{bidId} | 応募編集 |
| listMessages | GET | /api/bids/{bidId}/messages | 連絡履歴取得 |
| createMessage | POST | /api/bids/{bidId}/messages | メッセージ送信 |
| proposeFinalOffer | POST | /api/bids/{bidId}/final-offer | 最終条件提示 |
| agreeFinalOffer | POST | /api/bids/{bidId}/final-offer/agree | 合意する |
| discardFinalOffer | POST | /api/bids/{bidId}/final-offer/discard | 提示中の最終条件を破棄する |
| reportDeparture | POST | /api/jobs/{jobId}/departure-report | 運送開始報告 |
| reportCompletion | POST | /api/jobs/{jobId}/completion-report | 完了報告 |
