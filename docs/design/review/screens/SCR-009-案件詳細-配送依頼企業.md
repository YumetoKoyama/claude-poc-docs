# SCR-009 案件詳細画面（配送依頼企業）

## 要約

配送依頼企業が自社案件の編集・削除、応募者との連絡・最終条件提示・合意、完了確認までを行う中心画面。案件のライフサイクル全体（募集中〜評価済）を1画面で扱う。

対応ユースケース: UC-008, UC-009（編集・削除）, UC-014（連絡）, UC-015（最終条件提示）, UC-016（合意する）, UC-019（完了確認） / URL: `/jobs/{jobId}`（配送依頼企業）

---

## 画面遷移（入口と出口）

| 方向 | 画面 | 条件 |
|------|------|------|
| ← 入口 | SCR-008 案件一覧 | 案件行クリック |
| ← 入口 | SCR-011 取引履歴一覧 | 案件行クリック（読み取り専用） |
| ← 入口 | SCR-012 通知一覧 | 通知クリック（配送依頼企業） |
| ← 入口 | SCR-009-01 セット応募連動確認モーダル | 一括承認成功／キャンセル |
| ← 入口 | SCR-010 評価登録 | 登録成功後 |
| → 出口 | SCR-009-01 セット応募連動確認モーダル | セット応募の応募一覧クリック（対象がセット応募） |
| → 出口 | SCR-010 評価登録 | 完了確認後「評価する」（`status == COMPLETED`） |
| → 出口 | SCR-008 案件一覧 | 一覧へ戻る／削除成功 |

## 表示項目

### 入力

| 項目名 | 型 | 必須 | 送信先（operationId.フィールド） | 説明 |
|-------|----|------|--------------------------|------|
| 案件編集内容 | 案件登録.md の入力項目に準ずる | 編集時必須 | `updateJob.*` | 募集中・交渉中のみ活性 |
| 削除確認操作 | ボタン | — | `deleteJob` | 募集中・交渉中のみ活性 |
| メッセージ本文 | 文字列 | 送信時必須 | `createMessage.body` | 対象は選択中の応募 `bidId` |
| 最終条件（金額・場所日時・トラック種別） | 各項目 | 提示時必須 | `proposeFinalOffer.*` | |
| 合意操作 | ボタン | — | `agreeFinalOffer`（単体）/ `agreeSetBid`（セット） | |
| 破棄操作 | ボタン | — | `discardFinalOffer` | 提示中の最終条件を合意せず破棄（Q-J2、再提示不可）。単体応募のみ |
| 完了確認操作 | ボタン | — | `confirmCompletion` | |

### 出力・表示内容

| 項目名 | 型 | データ源 | 説明 |
|-------|----|--------|------|
| 案件基本情報・ステータス | オブジェクト | `getJobById.*` | |
| 応募一覧（会社名・金額・連絡先） | リスト | `listBidsForJob.items[]` | 配送依頼企業視点は全応募の実名・金額・連絡先（BR-020）を表示 |
| セット応募の連動案件表示 | リスト | `getSetBidById.bids[]` | 対象応募がセット応募の場合 |
| 連絡履歴 | リスト | `listMessages.items[]` | 送信者ユーザー名を表示 |
| 現在の最終条件提示状況 | オブジェクト | `getBidById.currentFinalOffer` | |
| 削除済み表示 | 真偽値 | `getJobById.logicallyDeleted` | |
| 成約スナップショット | オブジェクト（nullable） | `getJobById.contract` | 成約済以降のみ非null |
| 運送ステータス日時 | 日時群 | `getJobById.departureReportedAt` / `completionReportedAt` / `completionConfirmedAt` | |

> **⚠️ 制約・注意事項**: 成約済以降は編集・削除・メッセージ送信・最終条件提示のいずれも不可（MSG-021）。個人担当者連絡先は成約済以降のみ非null（BR-020）。合意操作は相手の状態変化により409（MSG-009、Q-J3）となることがある。

## 受け入れ条件

| AC-ID | 区分 | Given（前提状態） | When（操作） | Then（期待結果） | 関連 BR | operationId |
|-------|------|-----------------|------------|----------------|--------|-------------|
| 交渉合意成約/AC-001 | 正常系 | 交渉中の案件・応募 | 一方が最終条件を提示し相手が合意する | 案件ステータスが「成約済」に遷移し、成約スナップショットが保存され、他の応募者がクローズされ通知される | BR-012, BR-013, BR-014 | agreeFinalOffer |
| 交渉合意成約/AC-002 | 正常系 | セット応募が交渉中の状態 | セット一括で最終条件提示・合意が行われる | セット内すべての案件が一括で成約済みとなる | BR-008 | agreeSetBid |
| 運送ステータス報告/AC-002 | 正常系 | 案件ステータスが「運送中」で運送会社が完了報告済みの状態 | 完了確認操作を行う | 案件ステータスが「完了」に遷移する | BR-017 | confirmCompletion |
| 案件削除/AC-001 | 正常系 | 案件ステータスが「募集中」または「交渉中」の状態 | 削除操作を行う | 論理削除され、応募していた運送会社へ通知（MSG-013）が送信される | BR-022 | deleteJob |
| 交渉合意成約/AC-101 | 異常系 | 成約済み以降の案件・応募 | メッセージ送信や最終条件提示を試みる | 操作が拒否される | BR-016, BR-019 | createMessage, proposeFinalOffer |
| 案件削除/AC-101 | 異常系 | 案件ステータスが「成約済」「運送中」「完了」「評価済」のいずれかの状態 | 編集・削除を試みる | 操作導線が表示されず実行できない | BR-019 | updateJob, deleteJob |
| 運送ステータス報告/AC-102 | 異常系 | 運送会社が完了報告を行っていない状態 | 完了確認を行おうとする | 完了確認操作は表示・実行できない | BR-017 | confirmCompletion |
| 交渉合意成約/AC-201 | 境界値 | セット応募のうちいずれか1案件が他社に先に成約された状態 | 当該セット応募の状態を確認する | セット応募全体が自動的に不成立となり、セット内の他案件への応募も連動してクローズされる | BR-006, BR-015 | getSetBidById |
| 交渉合意成約/AC-301 | 権限境界 | 自社が当事者ではない案件・応募 | 連絡・最終条件提示・合意操作を試みる | 拒否される（404） | — | getJobById, createMessage, proposeFinalOffer, agreeFinalOffer, agreeSetBid |
| 運送ステータス報告/AC-301 | 権限境界 | 自社が当事者ではない案件 | 完了確認操作を試みる | 拒否される（404） | — | confirmCompletion |
| 交渉合意成約/AC-401 | エッジケース | 最終条件提示済み・未合意の状態で同一案件の別応募が先に成約 | 提示中の応募の状態を確認する | 当該応募は自動的にクローズされ「他社に決定しました」通知が送付される。提示中であった最終条件提示は無効化され、以後の合意操作は拒否される | BR-014, BR-012 | agreeFinalOffer |

## 対応 API

| operationId | メソッド | パス | 概要 |
|-------------|---------|------|------|
| getJobById | GET | /api/jobs/{jobId} | 案件詳細取得 |
| updateJob | PATCH | /api/jobs/{jobId} | 案件編集 |
| deleteJob | DELETE | /api/jobs/{jobId} | 案件論理削除 |
| listBidsForJob | GET | /api/jobs/{jobId}/bids | 応募一覧取得 |
| getBidById | GET | /api/bids/{bidId} | 応募詳細（最終条件含む） |
| listMessages | GET | /api/bids/{bidId}/messages | 連絡履歴取得 |
| createMessage | POST | /api/bids/{bidId}/messages | メッセージ送信 |
| proposeFinalOffer | POST | /api/bids/{bidId}/final-offer | 最終条件提示 |
| agreeFinalOffer | POST | /api/bids/{bidId}/final-offer/agree | 合意する（単体応募） |
| discardFinalOffer | POST | /api/bids/{bidId}/final-offer/discard | 提示中の最終条件を破棄する |
| agreeSetBid | POST | /api/set-bids/{setBidId}/agree | セット一括合意 |
| getSetBidById | GET | /api/set-bids/{setBidId} | セット応募詳細 |
| confirmCompletion | POST | /api/jobs/{jobId}/completion-confirmation | 完了確認 |
