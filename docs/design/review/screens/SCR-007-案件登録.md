# SCR-007 案件登録画面

## 要約

配送依頼企業が積荷1件分の案件（from/to場所・時刻、容積、物品種別、希望金額等）を新規登録する。登録直後は「募集中」ステータスになる。

対応ユースケース: UC-007（案件登録） / URL: `/jobs/new`

---

## 画面遷移（入口と出口）

| 方向 | 画面 | 条件 |
|------|------|------|
| ← 入口 | SCR-006 配送依頼企業ダッシュボード | 「案件登録」 |
| → 出口 | SCR-008 案件一覧 | 登録成功（`createJob` が 201） |
| → 出口 | SCR-006 配送依頼企業ダッシュボード | キャンセル |

---

## 表示項目

### 入力

| 項目名 | 型 | 必須 | 送信先（operationId.フィールド） | 説明 |
|-------|----|------|--------------------------|------|
| from 場所 | 文字列 | ○ | `createJob.fromLocation` | |
| from 時刻 | 日時 | ○ | `createJob.fromDatetime` | |
| to 場所 | 文字列 | ○ | `createJob.toLocation` | |
| to 時刻 | 日時 | ○ | `createJob.toDatetime` | |
| 容積 | 数値 | ○ | `createJob.volumeM3` | 単位 m3（Q-DM3） |
| 属性 | 複数選択 | 任意 | `createJob.attributes[]` | 危険物/要冷蔵/要冷凍/割れ物 |
| 物品種別 | 単一選択 | ○ | `createJob.cargoType` | |
| 希望トラック種別 | 単一選択 | ○ | `createJob.truckType` | |
| 希望金額 | 数値 | ○ | `createJob.desiredAmount` | 単位: 円（税別）、整数 |
| 備考 | 文字列 | 任意 | `createJob.remarks` | |

### 出力・表示内容

| 項目名 | 型 | データ源 | 説明 |
|-------|----|--------|------|
| 登録結果 | 案件ID・ステータス | `createJob.jobId` / `createJob.status` | 成功時 SCR-008 へ遷移 |

> **⚠️ 制約・注意事項**: from 時刻が to 時刻より後の場合は不整合エラー（MSG-001）。`fromLocation`/`toLocation`/`remarks` は XSS 対策として `pattern: '^[^<>]*$'` を適用（HTML タグ不可）。運送会社ユーザーは 403（配送依頼企業のみ登録可能）。

## 受け入れ条件

| AC-ID | 区分 | Given（前提状態） | When（操作） | Then（期待結果） | 関連 BR | operationId |
|-------|------|-----------------|------------|----------------|--------|-------------|
| AC-001 | 正常系 | 案件登録項目をすべて有効な値で入力した状態 | 登録操作を行う | 案件がステータス「募集中」で作成され一覧に表示される | BR-011 | createJob |
| AC-101 | 異常系 | 必須項目が未入力、またはfrom時刻がto時刻より後になっている等の不整合がある状態 | 登録操作を行う | MSG-001が表示され登録は行われない | — | createJob |
