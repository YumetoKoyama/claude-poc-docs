# SCR-002 企業アカウント登録画面

## 要約

配送依頼企業・運送会社いずれかの企業アカウント（テナント）と初回ユーザーを新規登録する。未認証で到達できる画面。

対応ユースケース: UC-001（企業アカウント登録・配送依頼企業）, UC-002（同・運送会社） / URL: `/register`

---

## 画面遷移（入口と出口）

| 方向 | 画面 | 条件 |
|------|------|------|
| ← 入口 | SCR-001 ログイン | 「新規登録」リンク |
| → 出口 | SCR-001 ログイン | 登録成功（`registerTenant` が 201） |
| → 出口 | SCR-001 ログイン | キャンセル / 戻る |

---

## 表示項目

### 入力

| 項目名 | 型 | 必須 | 送信先（operationId.フィールド） | 説明 |
|-------|----|------|--------------------------|------|
| 企業種別 | 単一選択 | ○ | `registerTenant.tenantType` | 配送依頼企業 / 運送会社 |
| 法人名 | 文字列 | ○ | `registerTenant.companyName` | |
| 住所 | 文字列 | ○ | `registerTenant.address` | |
| 会社電話番号 | 文字列 | ○ | `registerTenant.companyPhone` | |
| 会社メールアドレス | メール形式 | ○ | `registerTenant.companyEmail` | |
| 支払方法 | 単一選択 | ○ | `registerTenant.paymentMethod` | 請求書払い / 銀行振込 |
| 利用規約・プライバシーポリシー同意 | チェック | ○ | `registerTenant.termsAgreed` | |
| ユーザー名（識別名） | 文字列 | ○ | `registerTenant.initialUser.displayName` | |
| ログイン ID | 文字列 | ○ | `registerTenant.initialUser.loginId` | |
| 登録メールアドレス | メール形式 | ○ | `registerTenant.initialUser.email` | |
| ログイン用パスワード | 文字列 | ○ | `registerTenant.initialUser.password` | 最小8文字、英字+数字各1文字以上（Q-NF4） |

### 出力・表示内容

| 項目名 | 型 | データ源 | 説明 |
|-------|----|--------|------|
| 登録結果メッセージ | 文字列 | `registerTenant` の 201/400/409 | 成功時は SCR-001 へ遷移し案内表示 |

> **⚠️ 制約・注意事項**: 法人名＋住所が既存テナントと完全一致（正規化後、Q-A1）の場合は重複エラーとなり、救済手段（自動的な統合等）はない。既存テナント管理者への連絡を案内する文言のみ表示する（Q-A2）。DB UNIQUE 制約で最終防御する。

## 受け入れ条件

| AC-ID | 区分 | Given（前提状態） | When（操作） | Then（期待結果） | 関連 BR | operationId |
|-------|------|-----------------|------------|----------------|--------|-------------|
| AC-001 | 正常系 | 企業共通項目・初回ユーザー項目をすべて有効な値で入力し利用規約に同意した状態 | 登録操作を行う | テナントと初回ユーザーが作成され、ログイン画面へ遷移する | BR-001, BR-002 | registerTenant |
| AC-101 | 異常系 | 利用規約・プライバシーポリシーに同意していない状態 | 登録操作を行う | MSG-018が表示され登録は行われない | BR-001 | registerTenant |
| AC-102 | 異常系 | 必須項目が未入力、またはメールアドレス等の形式が不正な状態 | 登録操作を行う | MSG-014系の入力エラーが表示され登録は行われない | BR-002 | registerTenant |
| AC-201 | 境界値 | 入力された法人名＋住所が既存テナントの法人名＋住所と完全一致（正規化後）する状態（Q-A1） | 登録操作を行う | 重複エラーが表示され新規テナントは作成されない | — | registerTenant |
| AC-401 | エッジケース | 法人重複と判定された状態（Q-A2: 救済手段なし） | 登録操作を行う | 登録不可のみを表示し、既存テナント管理者への連絡を利用者に案内する | — | registerTenant |
