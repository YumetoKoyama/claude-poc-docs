# UI 設計ブリーフ テンプレート

UI 設計ブリーフは **2層構造** で作成する:

1. **共通ブリーフ**: `docs/design/ui-design/brief/_共通.md`
   - 全画面横断の DS・共通コンポーネント・ロール・トーン・状態規約・レスポンシブ方針
   - Claude Design に **最初に** 投入してデザインシステムを確立する
2. **画面別ブリーフ**: `docs/design/ui-design/brief/[scr-id]-[画面名].md`
   - 共通ブリーフからの **差分のみ** を記述
   - Claude Design には共通ブリーフ確立後に **まとめて添付** する

すべての項目に `参照:` で出典（設計 md / 要件 md）を必ず付ける。設計書に書かれていない事項は **新たに決めず**、最終節の「未確定事項」に逃がす。

### 略号・ID 採用時の凡例必須ルール

CLAUDE.md の開発ルールに従い、UI 設計ブリーフでも **ID または略号を導入する場合は、当該ドキュメントの冒頭付近に凡例を必ず出力する**。`_共通.md` でロール略号やコンポーネント略号（例: `CMP-001`、`Role-A`/`Role-B` 等）を新規導入した場合は `_共通.md` 冒頭の凡例セクションに掲載し、画面別ブリーフはその凡例を参照する形にする（画面別ブリーフ側で新規略号を導入しない）。

引用のみで導入しない既知 ID（`SCR-XXX`、`AC-XXX` 等）については、出典 md 側の凡例を `参照:` で示せばよい。

## ファイル名規約

`[SCR-ID]-[画面名].md`（例: `SCR-001-ログイン.md`、`SCR-003-アイテム一覧.md`）。SCR-ID は大文字、画面名は業務用語の日本語で記述する。
対応する設計 md と同じファイル名を使い、相互参照を容易にする。

共通ブリーフは `_共通.md` 固定（先頭アンダースコアでソート上位に来るため、目視で「共通」と分かる）。

## Claude Design 投入手順（順序固定）

1. claude.ai/design でプロジェクトを新規作成する。
2. **最初に `_共通.md` を添付** し、その末尾の「Claude Design 投入用プロンプト（共通投入時）」をチャットに貼る。Claude Design がデザインシステム方針を提案するまで待つ。
3. **画面別ブリーフ（`SCR-*.md`）を一括で（または役割/カテゴリ単位で）添付** し、「これらの画面を上記 DS で生成してください」と依頼する。
4. 生成結果を確認し、必要に応じて対話で調整する。
5. Export → **Handoff bundle** を取得し、**Export 構造そのまま** `docs/design/ui-design/handoff/` 配下に格納する（このディレクトリ作成と格納は人手で実施）。
6. handoff bundle の `README.md` にある画面 ID → prototype 関数のマッピング表が、後続の `create-issues-from-docs` の入力となる。

## ui-design/handoff の格納ルール（重要）

Claude Design の Export 物は **Export 構造そのまま** で格納すること。画面単位 (`[scr-id]/`) で切り分けない。

```
docs/design/ui-design/handoff/
├── README.md              ← Export の README（scr-id → prototype 関数のマッピング表を含む）
├── prototype/
│   ├── index.html
│   ├── wf-primitives.jsx       ← 共通プリミティブ（横断的共有資産）
│   ├── wf-shell.jsx            ← AppHeader / SideNav（横断的共有資産）
│   ├── wf-screens-auth.jsx     ← 認証系画面群（複数 SCR が同居）
│   ├── wf-screens-shipper.jsx
│   └── ...                     ← カテゴリ単位の jsx
└── tokens/
    ├── colors_and_type.css
    └── DESIGN_SYSTEM_GUIDE.md
```

prototype/ 配下を人手で画面単位に分割してはいけない。共通プリミティブ・シェル・トークンは横断的共有資産であり、画面 ID 単位に分割すると同期地獄になる。画面 ID と prototype 関数の対応は `ui-design/handoff/README.md` のマッピング表が Source of Truth。

## デザインシステムの優先順位

- Claude Design 側に組織デザインシステムが構築されている場合 → そちらを優先。
- 構築されていない、または上書きが必要な場合 → `_共通.md` の値を共通ブリーフ投入時に明示し、Claude Design にも貼り込む。
