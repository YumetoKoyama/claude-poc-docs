# 非機能要件 要求値記述テンプレート

「方針」ではなく「要求値」として記述するためのテンプレート。
空欄の場合は `[要確認]` を記入し、オープン課題に転記する。

---

## 1. 性能要件

| 対象 | 要求値 | 計測条件 |
|------|--------|---------|
| 一覧画面の初期表示 | __ 秒以内 | 同時接続 __ ユーザー時 |
| 検索・絞り込み | __ 秒以内 | 同上 |
| 更新・登録操作 | __ 秒以内 | 同上 |
| ファイルアップロード | __ MB まで、__ 秒以内 | |

---

## 2. 可用性・稼働要件

| 項目 | 要求値 |
|------|--------|
| 稼働時間帯 | 例: 平日 9:00〜22:00 / 土日 9:00〜18:00 |
| 計画停止方針 | 例: 毎週水曜 3:00〜4:00 をメンテナンス枠とする |
| 月次稼働率目標 | 例: 99% 以上（計画停止時間を除く） |
| 非計画停止の目標 | 例: 月1回以内 |

---

## 3. バックアップ・復旧要件

| 項目 | 要求値 |
|------|--------|
| バックアップ取得頻度 | 例: 1日1回（深夜2:00） |
| バックアップ保管期間 | 例: 30日間 |
| 復旧目標時点（RPO） | 例: 前日のバックアップ時点まで |
| 復旧目標時間（RTO） | 例: 4時間以内 |

---

## 4. 障害対応・監視要件

| 項目 | 要求値 |
|------|--------|
| 重大障害（P1）の検知・通知時間 | 例: 発生から15分以内に担当者通知 |
| 通知先・連絡経路 | 例: SlackチャンネルXX、担当者A→エスカレーション先B |
| 監視対象 | 例: サーバー死活・レスポンスタイム・エラーレート |
| 対応責任範囲 | 例: アプリケーション層は開発チーム、インフラ層はインフラチーム |

---

## 5. ログ要件

| 項目 | 要求値 |
|------|--------|
| ログ保管期間 | 例: アクセスログ 90日、エラーログ 1年 |
| 記録対象操作 | 例: ログイン・ログアウト・案件作成・削除・成約・管理者操作 |
| 個人情報のログ出力 | 例: 氏名・メールアドレスはマスキングする |

---

## 6. セキュリティ要件

| 項目 | 要求値 |
|------|--------|
| パスワード最低文字数 | 例: 8文字以上 |
| パスワード文字種要件 | 例: 英大文字・英小文字・数字をそれぞれ1文字以上 |
| パスワード有効期限 | 例: なし（強制変更なし）または __ 日 |
| ログイン失敗ロック | 例: 5回連続失敗で30分ロック |
| セッションタイムアウト | 例: 最終操作から30分 |
| 同時ログイン | 例: 許可する / 後発セッションを弾く / 両方に通知して選択させる |
| 通信暗号化 | 例: 全通信をHTTPS必須（HTTP→HTTPSリダイレクト） |

---

## 7. 対応環境要件

| 項目 | 要求値 |
|------|--------|
| 対応ブラウザ | 例: Chrome最新版、Edge最新版、Safari最新版 |
| 対応OS（PC） | 例: Windows 10以上、macOS 13以上 |
| 対応デバイス（スマートフォン） | 例: iOS 16以上、Android 13以上 |
| 画面サイズ | 例: PC: 1280px以上、スマートフォン: 375px以上 |
| JavaScript無効時の動作 | 例: エラーメッセージを表示する（SPA前提のため動作不可） |

---

## 8. アクセシビリティ要件

| 項目 | 要求値 |
|------|--------|
| 準拠基準 | 例: WCAG 2.1 AA 準拠 / 非対応（デモ用途のため） |
| 色コントラスト | 例: WCAG AA基準（4.5:1以上） |
| スクリーンリーダー対応 | 例: 対応する / 対応しない |
